Security Operator

This screen is used to define user IDs and their access profile.

Label

Description

Operator

User ID (used when logging in to LIFEfit)

This is used to "stamp" any transactions created by this user, as part of the audit trail.

Users should not share IDs as this destroys the auditability of transactions.

Name

User Name

Password

The password is hidden.

As a security measure, if you want to change it, you will have to enter your own LIFEfit password first.

Telephone No

User phone number

Active user

Defines whether a user is allowed to access LIFEfit.

This is typically unchecked when a user leaves the organisation, or moves to a job where LIFEfit is not used.

It is automatically unchecked if the user exceeds the maximum number of failed login attempts (usually when they have forgotten their password). In this case, the system administrator can make them active again, and reset the password if necessary.

System Admin Auth

Does the user have system administrator privileges?

e.g.

- allowed to enquire on policies for all companies and branches.

- allowed to maintain user profiles.

Allow access via SOAP

Allow web services access through this user

Allow Access After Batch Run

Tick if the user is allowed to log in when the system is in batch check status. This allows users without full system administrator privileges to undertake post batch checks.

Menu

Select the menu that this user sees.

This setting overrides a default set at "user group" level.

Language

User's language of preference for screen display.

If language translations for this language have been entered for any screens, these will be shown instead of the default text.

This can be also be used for company-specific terminology (e.g. face amount / sum assured) as well as for different national languages.

Branches

Which branches' policies is this user allowed to access.

Security Group

User group to which the person belongs.

Certain attributes are defined by the group to which the user belongs - although most can be overridden on the user ID.

Position

Job Title

E-Mail

User e-mail address.

Used in policy diary reminder functionality (the reminder is set up for a user ID, and the e-mail is sent to this address)

Path to Document Server directory (if user-specific)

Enables a printer location to be user-specific.

The default printer path is defined at system level. It can be overridden at user group level, and again at individual user level.

This is useful where different documents are printed by different categories of user in different locations.

Payments Out Limits - payout limit

(one value for each currency defined in LIFEfit)

Maximum payment out amount that the user can initiate, and will be paid without requiring second user authorisation.

If set to zero, all payments will require second user authorisation.

Payments Out Limits - Payout authorisation limit

(one value for each currency defined in LIFEfit)

Maximum payment out amount that the user can authorise, where the payment was initiated by someone else.

Typically a low amount is set for clerical users, and higher amounts for supervisors and senior managers, depending on company rules

e.g.

claim administrator can authorise up to 10,000

claims supervisor can authorise up to 50,000

operations director can authorise up to 250,000

managing director can authorise up to 5,000,000

n.b. LIFEfit maintains an audit trail of who initiated a claim, and who authorised it (including time and date of action).
