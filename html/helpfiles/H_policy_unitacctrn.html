Unit Accounting

Lists the unit accounting transactions for a policy

Records are sorted in reverse chronological order by default. You can click on any of the highlighted column headings, to sort in the order of that column. Click the heading again to reverse the sort order.

Click on an individual transaction in the list to see full details

Label

Description

Created Date

Server date when the transaction was created

Eff Date

Effective date of the transaction

Reason

Reason for the transaction.

From CODES record UNIT_REASON.

Each reason has a different set of accounting postings associated, which all follow a common pattern.

Coverage

Policy coverage to which the transaction relates

Unit Date

Unit price date used in the transaction

Trans.No

Transaction number

Fund

Funds involved in the transaction

Source

If the transaction was created as an adjustment, UAD is shown here
