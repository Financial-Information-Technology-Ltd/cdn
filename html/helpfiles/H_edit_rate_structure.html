Rate Structure Setup

Rate tables are used in many places in LIFEfit, for example premium rate tables, unit-linked allocation rate tables and surrender penalty rate tables.

The multi-part key structure of rate tables is defined here; LIFEfit supports an 8-part key, which offers great flexibility in rate table definition.

The rates themselves are entered (or imported) at Support > Rates > Rate Entry. They will conform to the key structure defined here.

Version Date

Rate Version Date

LIFEfit supports multiple, dated versions of the rates.

This enables new rates to be used for existing product types. It also enables, in conjunction with Date type above, policies issued (or applied for) on different dates to use the rates applicable at that date.

E.g. annuity rates can be held as at the application date, even if the rate for new applications changes.

Enter a date in the format dd/mm/yyyy, or click the calendar icon and choose a date. Rates will be used from that date onwards

Version No

Rate Version Number

The oldest dated version should be 1, the next 2, and so on in chronological order.

e.g.
