Periodic Processing Groups

Set up screen for batch processes

Label

Description

Code

Batch process code, as entered or selected on the previous screen. Display only.

Title

Free format text.

Allow only one process to run at a time

Tick if the batch must be run exclusively (no other batch process can be run concurrently).

Batch setup section

Type

Select from the dropdown list:

       P Process (a program)

       M Menu (spawns another batch process)

Process Name

If type P selected, choose from the dropdown of programs.

If type M selected, choose from the dropdown of other batch processes

Run

Tick if this process / menu should be run.

This feature is primarily used in testing, to enable individual programs to be run. In production, it is usual that all processes in a batch are switched on.

Parameter

Enter the parameter(s) required by the batch program, if any.

e.g. batch program LF.REPORT.RUN takes a parameter of the report name.

<Insert> button

Insert a process in the list.

The sequence in which batch processes are run is important; a process may be logically dependent on the result of a process ran earlier e.g. commission calculation must be run after processes which pay premiums.

<Delete> button

Delete a process from the list.

<Add> button

Add another program or menu to the list.

<Save> button

Save changes.

<Return> hyperlink

Return to the main menu.
