Query Editor

This screen allows you to create and amend ad hoc LIFEfit reports, sometimes referred to as "Davywriter" reports.

Reports may be run online (at Support > Reports > Run Reports), or called by batch job LF.REPORT.RUN using the report name as a parameter.

Online reports can be produced in CSV format (e.g. for Excel), HTML format (web page) or fixed-with text. Batch reports are produced in CSV format, and are often used for management information or audit purposes.

In addition, report selections defined here can be used as criteria for mailshot selection (at Support > Mailshot).

You can produce reports with table joins (foreign key links), although for sophisticated reports with multiple levels of table joins you may like to consider exploiting LIFEfit's ODBC compatibility. You can utilise specialist reporting tools such as Query for MS Excel, MS Access, Business Objects and Qlikview. If you need this type of reporting, please ask your system administrator to set up your PC for ODBC access to the LIFEfit dataset.

Repeat multivalues

LIFEfit's U2 database supports multivalues. Tick this box if you want single value fields to be repeated for each multivalue - this can make data easier to analyse, for example in an Excel pivot table.

Examples of multivalue fields are plan code, issue date and mode/method on the policy file. They are "multivalue by coverage" (i.e. repeated for each coverage on a policy), whereas the policy number is single value.

Display fields heading

Display field

Either enter the field name, or click "Dictionary" and pick a field from the list of all fields on the file.

These will be the columns in the report. The column heading is either the "raw" field name, or the description from CODES record DIXT_xxxx (where xxxx is the file name) if it exists. This CODES record can be extended by the user and the description is also seen on the "pick list".

As well as data fields on file, you can also include any symbolic fields that are available for documents for this file.

e.g.

- GET.FACE.AMOUNT.SYMB returns a policy's face amount (which may be calculated as a percentage of policy value)

- GET.NEXT.PREMIUM.AMOUNT.SYMB returns a policy's next premium (which may include indexation)

- LF.OWNER.FIRST.NAME.SYMB returns the policy owner's first name

- LF.OWNER.SURNAME.SYMB returns the policy owner's surname

- IS.POLICY.ASSIGNED.SYMB returns boolean to say if the policy is assigned (1 = yes, 0 = no)

Report Availability Restrictions heading

Some reports contain sensitive information, for example workflow statistics, so LIFEfit has user level security around access to reports. This is why "report definition" and "report running" are items on the menu.

Report Owner

User who created the report

Security Groups

User groups that are allowed to "see" the report. If they have access to the "create reports" menu item, users in these groups can amend the report. If not, they can run the report but not change it.

"Submit" Button

Saves the report. It will then be available from the "Run Reports" screen, subject to availability restrictions (described above).

"Save and Run" Button

Saves the report, and runs it. The report is generated in CSV format (e.g. for MS Excel).

tip: for large reports, it can often be quicker to open them directly from the spool (&hold&) directory on the server, rather than clicking to open them via the browser.

After running the report, the "return" button returns the user to this (report creation) screen.

"Delete" Button

Deletes the report. This button is only available to the report owner.

"Return" Hyperlink

Returns you to the "report selection" screen.
