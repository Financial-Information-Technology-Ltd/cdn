Outstanding/Underwriting Requirements

Set the features of a new business underwriting requirement.

These are typically medical checks that need to be undertaken before the insurance company is prepared to put a policy on risk. As a general rule, the higher the sum assured and the older the applicant, the more medical checks will be required.

They are also used for financial checks (e.g. have we completed source of funds analysis?) and procedural checks (e.g. have we received the signed application form?) that must be undertaken before a policy can be issued.

The appropriate checks are applied automatically when a policy is created at new business, according to the thresholds (e.g. age, sum assured level) defined on the plan. They are cleared through policy maintenance.

Label

Description

Outstanding Code

Underwriting code, as selected or entered on the previous screen. Display only.

Description

Free format text input for description of the underwriting requirement e.g. ECG Tracing (Resting & Exercise)

Prevent status change if unresolved

If ticked, the policy cannot go active (on risk) until this underwriting requirement has been cleared.

Prevent exceeding free cover limit if unresolved

Used in group business, where a member certificate can go active before an underwriting requirement is cleared.

This enables companies to issue group life schemes where some members have not yet undergone medical tests, but limit the claims they are prepared to pay to the amount of cover that would be offered without medical tests being undertaken. Once a members underwriting requirements have been cleared, full cover (above the free cover limit) is offered.

Terminate cover when rejected

If ticked, life cover terminates when the underwriting test is failed (the applicant is considered too high a risk).

Often used for additional covers on group scheme business the member maintains the base level of life cover, but additional riders are not maintained (e.g., critical illness, permanent and total disability).

Created for whole policy (not coverage-specific)

If ticked, only one instance of the requirement is created per policy, even if it applies to more than one of the policys coverages.

Created for a whole scheme (group-business only)

If ticked, only one instance of the requirement is created per scheme, even if it applies to more than one of the schemes members.

Payment Required

Tick if the insurance company is required to pay for the test. A payment is made to the medical provider / clinic who undertake the test on the insurance companys behalf.

Number of days reply valid after received

The number of days after receipt of a test result that it is considered valid

Event (list type) generated when created

Document trigger to be created when the requirement is generated. This can be used to trigger a letter and/or email to the applicant, advising them to get a test, as well as to the clinic advising them that the applicant requires a medical test.

Enables follow-up (reminder) letters and emails, in the event that tests have been requested but results have not yet been received.
