Group Business Product parameters

Settings for products administered on a group scheme basis.

Label

Description

Plan Code

Plan Code display only

Plan Name

Plan Name display only

Stabilization fund

Tick if product uses stabilization fund.

The stabilization fund can used for deposit administration savings products, whereby a proportion of the deposits on all certificates is kept as a savings value on the group master policy. The proportion is defined on the group when it is created.

If a death claim amount exceeds the savings value of a certificate, the balance is paid from the group's stabilization fund.

Postpone member update until premium payment

Set this parameter so that endorsements are applied to certificates by the Pay Leavers/Joiners function.

Source of funds

Select whether this premium is paid by (therefor billed to) the employer or the employee from the dropdown list.

Invoices will show the split between employer and employee due premium.

Certificate key structure

Select the key structure for certificate numbers from the dropdown list

Options are:

         0 - PolicyKey_Number (i.e. all certificates are MasterPolicyNumber_MemberNumber)

         1 - As normal policy (i.e. sequential numbering, no way to identify as a certificate by looking at the number)

File Layout produced by calculate bill

Select the format for the file generated on clicking calculate bill (columns in the CSV file) from the dropdown list.

Options are:

         1 - Certificate No, Member, Coverage, Employer, Employee, Sum Assured

         2 - Certificate No, Member, Coverage, Premium, Sum Assured

         3 - Certificate No, Member, Coverage, Sex, Date of Birth, Salary, Premium, Sum Assured

         4 - Certificate No, Member, Life Assured, Coverage, Sex, Date of Birth, Salary, Premium, Sum Assured

         5 - Certificate No, Member, Beneficiary, Member Date of Birth, Coverage, Premium, Sum Assured

Premium payment is optional for group members

Tick if premium payment is optional (i.e. it is OK to pay premium on some certificates, but not others).

Premium is paid starting with the earliest paid to date

Tick if premium is paid starting with the earliest paid-to date.

Bill required when reconciling bank payment

Tick if it is obligatory to match a payment with the total amount requested on a bill (premium can then be reconciled to certificates as detailed on the bill).

Process to import membership information

Select the process used to import membership information from the dropdown list.

Process to export membership information

Select the process used to export membership information from the dropdown list.

Do not recalculate sum assured and premium on certificate after issue

Tick is premium and sum assured on a certificate should remain unchanged after issue. By default, they will be recalculated according to product rules.

Allowed Sum Assured basis

Select which sum assured bases are permitted from the dropdown list.

Options are:

         1 - Multiple of Salary

         2 - Fixed Amount

         3 - Multiple of Allowances

         4 - Multiple of (Salary + Allowances)

The user will choose from the options selected when defining the group and subgroup rules as part of scheme setup.

<Add> button

Add another sum assured basis

Allowed Sum Assured Multiplicators

For sum assured bases involving a multiple (1, 3 and 4 above), enter a permitted multiplier.

The user will choose from the options selected when defining the group, subgroup or member rules as part of scheme setup.

<Add> button

Add another permitted sum assured multiplier

Allowed Premium basis

Select which premium bases are permitted from the dropdown list.

Options are:

         1 Percentage of Salary

         2 - Fixed Amount

The user will choose from the options selected when defining the group and subgroup rules as part of scheme setup.

<Add> button

Add another premium basis

Allowed Premium Percentages

For premium basis 1 above (Percentage of salary) enter a permitted salary percentage.

The user will choose from the options selected when defining the group, subgroup or member rules as part of scheme setup.

<Add> button

Add another allowed premium percentage.

Create rider for dependants

Tick if riders should be created for each dependant benefit. (e.g. separate health benefit riders fro spouse and each child)

Minimum number of members

Enter the minimum number of group members permitted for a group scheme.

Maximum number of members

Enter the maximum number of group members permitted for a group scheme.

Keep Sum Assured above Free Cover Limit for initial period (days)

Enter the number of days after which, if underwriting is unresolved, a members sum assured should be automatically reduced to the free cover limit for the group.

For leavers and joiners use calculation based on annual premium

Set this parameter so the system calculates the required adjustment to the annual premium.

Create a Web User for the group owner when group is issued

Tick if a web security (eSurance) id should be created for the group owner when the scheme goes active.

Member can choose from all riders available on plan

Tick if a member can choose from all available riders on a plan.

What should happen on group surrender

Select the desired processing on group surrender form the dropdown list.

Options are:

         G - Surrender and pay to group owners

         H - Leave certificate with value is paid-up state

         M - Surrender and pay to group members

Days before the renewal date when the renewal process starts

Enter the number of days before the renewal date that the renewal process should be triggered. This determines the timing of the REN future record on the group master policy.

Use new country at group renewal

Tick if members should be re-rated based on their country of residence at group renewal. If unticked, the country of residence at issue will be used.

Create Workflow at scheme renewal

Tick if workflow task GROUP_RENEWAL should be created at group renewal.
