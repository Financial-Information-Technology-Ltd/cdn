Plan Cessation

This screen contains parameters that determine the rules for maturity (savings coverages) and expiry (risk coverages).

Maturities are automatically triggered by use of a MAT future record, processed by batch job LFS.AUTO.EXPIRY. Expiries are triggered by an EXP future record which is processed by the same batch job.

Although the MAT or EXP future record is created for the maturity/expiry date on policy issue, it is only displayed in the policys future record enquiry screen during the last year of its term. If the policy term is endorsed, the future record is moved to the new policy maturity/expiry date.

Maturity type

Select the way in which the maturity date for savings policies is set from the dropdown list.

Options are:

         A - Specified age.

         W - Whole Life (i.e. never matures - runs until death or surrender)

         Y - Specified term of policy.

Maturity at (years)

Policyholder age at which policy matures. Often used for pension savings policies.

Maturity at for females (if different)

Policyholder age at which policy matures for female policyholders (if different from the age for males). Often used for pension savings policies with gender-specific rules.

Maturity months

Number of months to maturity, used in conjunction with <Maturity at (years)> if <Maturity type> = "Y".

Maturity days

Number of days to maturity, used in conjunction with <Maturity at (years)> and <Maturity months> if Maturity type = "Y".

e.g. Maturity term can be set to 4 years, 3 months and 14 days by the following:

         Maturity at (years): 4

         Maturity months: 3

         Maturity days: 14

Maximum maturity age

Maximum policyholder age at maturity date.

This is checked at new business entry

i.e. if (issue age + term) > maximum maturity age, an error will be returned.

Minimum maturity age

Minimum policyholder age at maturity date.

This is checked at new business entry

i.e. if (issue age + term) < minimum maturity age, an error will be returned.

Maturity calc code

Defines how the policys maturity value (paid to the policyholder on maturity) is calculated.

Options are

         0 - Value of the policy (used for unit-linked or deposit administration savings)

         1 - Sum Assured - Unpaid premiums

         2 - Sum Assured

SA Calc Code

Sum assured calculation code (used in conjunction with Maturity calculation codes 1 and 2 above).

Options are:

         0 - As per Face Amount

         1 - Greater of (FA, Bid value of units)

         2 - Bid vals (init units * curr age/100)+(acc units - plan debt)

         3 - Bid val of acc units * acc unit surr factor

         4 - Value of Units in Notional Fund

         5 - Dependent on policy Face Amt Flag on inclusive/additional basis

         6 - FA - [(t - 1) * FA / n]

         J - Face Amount + Single Premium

Maturity notice weeks

Number of weeks in advance of the maturity date that a maturity notice (letter and/or email) is to be automatically triggered.

It is standard practice to send a maturity notice to the policyholder in advance of the maturity date, so they can return the necessary documentation such as a copy of the original policy document and details of to whom and how the maturity amount should be paid.

The maturity notice is triggered by an MN future record, processed by batch job LF.GENERATE.MATURITY.NOTICE.

The MN trigger is also often used to initiate a workflow task, so the preparations for the maturity can be done in the back office. This usually entails checking that all supporting documentation has been received, and chasing for it if necessary.

Product bought at maturity

If this is a savings product which is used to purchase an annuity at maturity, select the name of the annuity product here from the dropdown list of plan codes.

This functionality is used in pre-sales projection calculations only, to show what annuity could be purchased assuming the projected maturity value. There is no processing to automatically create the annuity on policy maturity this can be done using standard new business processing and accounting for the premium on the annuity from the maturity value of the savings.
