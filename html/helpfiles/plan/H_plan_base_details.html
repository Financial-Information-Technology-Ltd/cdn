General

Free format text

Policy Number Digits

Number of digits in the policy number.

A policy's number is taken from field "Next policy number" on the company (Support files > Organisation > Company). This will be prefixed by leading zeros if necessary.

Policy Number Prefix

The prefix to be added to the system generated policy number.

Application Number Digits

Number of digits in the application number, used when a pre-sales application is generated in eSurance.

This will be prefixed by leading zeros if necessary.

Application Number Prefix

The prefix to be added to the system generated application number.

Use application number as policy number

For business that is initiated by a pre-sales application in eSurance (broker/agent web portal), this enables the application number to be used as the policy number when a policy record is created in the back office.

This feature can facilitate subsequent servicing as the customer can refer to his policy by the application number on pre-sales documentation generated online.

In addition, for non-underwritten business where the policy issue document itself is generated in the broker's office using eSurance, it enables the document to contain the policy number.

Do not record status change events

Determines whether policy events (e.g. document triggers on policy status change) are generated for policies of this plan type

Allowed for Group Business

Determines whether this plan can be sold on a group basis.

If checked, the plan will appear in the "product" dropdown when setting up a new group scheme.
