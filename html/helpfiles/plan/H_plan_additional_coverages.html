Plan Additional Coverages

For base plans, this screen is used to define which riders are available, and how and when they can be added to a policy.

For rider plans, this screen is used to determine their behaviour and relationship with any other coverages on a policy.

Label

Description

Plan Code (display only)

The plan code.

Plan Name (display only)

The plan name.

"Additional Coverages Details" section (used when the Plan Code is a base plan)

Rider Control

Select plans to be available as riders from dropdown list.

Dropdown is comprised of all plan codes known to LIFEfit

Mandatory

Determines whether policies must have this rider

Auto Rider

Determines whether this rider is added automatically in the new business process

Prevent New bus.

If checked, the rider cannot be selected at new business, only post-issue.

Rider Depend.

Enables you to control the availability of a rider, dependent on whether another rider has been selected. Enter the position in the list of the other rider that must be present for this rider to be allowed.

e.g . critical illness cover is only available if life cover has been chosen.

Maximum Number

Sets the maximum number of riders of this plan type that can be added to a policy.

Typically 1 for risk benefit riders, and many for investment (savings) riders such as single premium top-ups.

If left blank, there is no limit.

Available Post Issue

Determines whether (and when) a rider can be added to an active policy.

Select from dropdown

-           Available (i.e. at any time after policy issue)

-           Not available

-           Available at anniversary

This dropdown is from CODES record AVAILABLE_POST_ISSUE which cannot be extended by the user.

"How this plan behaves when rider" section (used when the Plan Code is a rider)

Avail as rider

Checked if this Plan Code is sold as a rider

Available as a rider only when policy has a death cover

Enables you to restrict the availability of this rider for policies whose base coverage has no death benefit. Typically used for additional death benefit riders.

How Rider collected

Defines how the premium is paid.

Select from dropdown

-           Does not apply to this plan (for free riders)

-           Collected modally (according to rider mode/method)

-           Deduct cost monthly from units with COI (include in monthly mortality charge, taken by cancelling units)

-           Deduct cost monthly from units separate from COI (deduct premium monthly by cancelling units, separately from mortality charge)

-           Deduct cost modally from policy fund (cancel units according to premium frequency, triggered by a UDP future record)

This dropdown comes from CODES record RIDER_COLLECT_TYPE which cannot be extended by the user.

Lapse when ratio of funded premium to total premium on policy exceeds this percent

For coverages whose premium is paid by unit cancellation, this determines the maximum percentage of units that can be cancelled. When this limit is reached, the rider Lapses.

Dealloc Start Dt Basis

For coverages whose premium is paid by unit cancellation, this determines when the first premium will be taken.

Select from dropdown

-           As standard (first premium taken on policy issue date)

-           Apply Alloc Price Dt Basis (uses the "UPP" date for the base coverage, which may be different from the premium paid date. Defined at Plans > Unit Linked Basic Details)

-           End of month (last working day of calendar month)

This dropdown comes from CODES record CHGS_START_DT which cannot be extended by the user.

Use Base Funds and Direction

Only applicable for riders with unit-linked investment.

If checked, this rider will use the same unit-linked funds and fund direction percentages as the base coverage.

If unchecked, the "new business" and " add rider " processes will request the funds and fund direction for this coverage.

Prevent rider if substandard

Enables the company to prevent a policy being activated with this rider if a mortality rating has been applied. Rider must be removed or terminated before the policy can be issued.

Sum Amount equals Base coverage Sum Assured

Often used for additional death or critical illness benefits

Copy Sum Assured from coverage with plan

Select plan code from dropdown. Used where there are multiple risk benefit riders whose sums assured are related.

Used in conjunction with "Percentage of Sum Assured from another coverage" below

Percentage of Sum Assured from another coverage

Used in conjunction with " Copy Sum Assured from coverage with plan".

Used where there are multiple risk benefit riders whose sums assured are related.

e.g . Critical illness cover is 75% of death benefit cover.

Decrease face amount pro rata to base coverage if base coverage is decreased due to free cover limit

Enables rider sum assured (also known as "face amount") to be automatically reduced if the base coverage sum assured is restricted due to the free cover limit.

Often seen in Group business.

Exclude this coverage from the Total Sum Assured when checking Maximum Sum Assured

Set this flag to exclude the Sum Assured from the Total Sum Assured when checking the Maximum Sum Assured
