Plan Escalation

Premium and benefit (sum assured) escalation settings for the plan

Label

Description

Allow escalation

Tick if premium and/or benefit escalation are allowed for policies of this plan type

Premium Frequency

Select premium escalation frequency from the dropdown list.

Options are:

         A - annually

         S - semi-annually

         Q - quarterly

         M monthly

A BII future record will be created for the premium escalation date, and the billing and premium payment processes will factor this in. When paid, the new premium value will be updated on the policy

Premium Basis

Select the premium escalation basis from the dropdown list.

Options are

         Compound

         Simple

         Recalculate premium

         Dynamic (compound)

Premium Rate

If the premium indexation rate for all policies of this plan type is the same, enter it here.

If not, the indexation rate will be set for the each individual policy at new business or post-issue endorsement.

Premium Index

If the premium is indexed using a published index, such as RPI, select the index here

Plan for "Slicing" escalation

Premium slice indexation involves the automatic creation of a new rider for each annual incremental increase in premium.

This is often used where the indexed premium amount is to be treated as new money in terms of agent commission, and (for unit-linked savings policies) its allocation rate

If this type of premium indexation structure is required, select the plan code to be used for annual premium slices from the dropdown list.

Benefit Frequency

Select benefit (sum assured) escalation frequency from the dropdown list.

Options are:

         A - annually

         S - semi-annually

         Q - quarterly

         M monthly

A BIIB future record will be created for the benefit escalation date, and the billing and premium payment processes will factor this in. When paid, the new sum assured value will be updated on the policy.

Benefit Basis

Select the benefit escalation basis from the dropdown list.

Options are

         Compound for face amount

         Simple for face amount

         Simple, rate = prem.esc.rate*ben.esc.rate

         Compound, rate = prem.esc.rate*ben.esc.rate

Benefit Rate (percent)

If all policies sums assured are to escalate at the same, fixed rate, enter the rate here

Benefit Index

If all policies sums assured are to escalate according to a dated table, select the table from the dropdown list.

The list is comprised of interest rate tables defined at Support > Rates > Interest Rates

This feature is often used if sum assured is escalated according to a published index such as RPI (retail price index).

Maximum Benefit Multiplier

Used to limit the maximum sum assured, in relation to the original sum assured at policy issue

e.g. a value of 2 will prevent sum assured escalation from increasing the sum assured to more than twice the initial sum assured.

Recalculations when escalating benefit

Defines premium recalculation required (if any) on sum assured escalation

Options are:

         No recalculation required

         Recalculate premium using Max Sum Assured

         Increase premium at the same rate as Sum Assured

Fee Basis

Escalation fee basis select form dropdown list

Option are

         % Deducted from all premiums paid

         % of value of premium increase

         % of increased annualised premium

Fee Rate

Escalation fee rate (if a fixed percentage)

Fee Table

Escalation fee table (if fee is set according to a rate table).

Select from the dropdown list of rate tables

Commission Basis

Select the escalation commission basis from the dropdown list

Options are

         % of premium increase

         % of Gross Modal Premium

        

Commission Rate

Escalation commission rate (if a fixed percentage)

Override Commission Table

Escalation commission rate table (if commission rate set according to a rate table)

Send letter before next indexation (number of days)

Number of days before the premium increase date that a letter is to be sent advising the policyholder of the upcoming increase.

An IXL (Indexation Letter) future record is created for this date.

The IX future record is processed by batch job LFS.FUTURE.IXL - Indexation Letters

Do not recalculate premium/face amount after escalation

Tick this option if premium indexation has no impact on sum assured, or if sum assured escalation has no impact on premium due.

Allowed Sum Assured Escalation Rates
