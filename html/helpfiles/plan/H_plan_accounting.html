Asset Accounting

Tick if an 'Issuable Cash' accounting entry should be created when premium is paid, in addition to a regular cash / premium entry.

The amount posted to pseudo ISSCASH will show how much of the premium received is available for investment (i.e. after risk premiums have been paid)

Accrual Accounting

Tick if accrual accounting is required for the plan.

Accrual accounting (also known as premium income accounting) is typically used for risk coverages.

If ticked, an ACC future record is created for the coverage on its issue date. This is processed in batch.

In year 1, the posting is:

dr REGFP (premium received)

cr PDOFN (premium due)

In subsequent years, the posting is:

dr REGRP (premium received)

cr PDOFN (premium due)

As such, premium income is posted when it is due, rather than when it is received.

On receipt of premium in year 1, the posting is

Dr CASH

Cr REGFP (premium received)

In subsequent years, thee posting is:

Dr CASH

Cr REGRP (premium received)

So, if premium is received as expected, the net posting is

Dr CASH

Cr REGFP (or REGRP)

On lapse, termination or death, any accrued premium which has not yet been received is reversed.

Accrual Accounting Frequency

Frequency of accrual accounting (determines the frequency of the ACC future record).

Helps to avoid overstating premium income (e.g. if premium is received annually, it can be accrued monthly to match the companys liability)

Accounting Data Pseudo Code

Each financial transaction in LIFEfit posts credits and debits to a set of accounting pseudo codes.

Mapping accounts at plan level enables detailed analysis by plan type (e.g. to analyse premium income by product type).

Enter a valid LIFEfit Accounting Pseudo Code, or press the search button to select one from a list. The description of the selected pseudo is shown on the right.

If a pseudo is not mapped here, LIFEfit will look for a mapping at company level (support > accounting > accounting specifications)

Account Number

Enter your own internal chart of account number to be used for transactions that are generated for the selected LIFEfit pseudo code. This ensures that accounts will be produced according to your general ledger chart of accounts.

This Account Number is the default for the system. Account Numbers can also be set at Plan level. Account Numbers set at Plan level will be used instead of this Account Number.

Plan level account numbers can be set using: Support Files > Product Workbench > Basic Accounting.
