Plan Claim-Related parameters

Screen Description

Label

Description

Date used in claim benefit calculations

Specifies which date will be used for benefit payable calculations.

Claim Notify Option

Specify what happens when a claim is notified (change to status D death notified). Options are:

       Calculate Benefit and Post Estimates - this is the standard option for non unit-linked products e.g. term and endowment. It ensures the technical reserve accounting posting is made.

       Calculate Benefit and Cancel Units - only applicable for unit-linked savings products.

       Calculate Benefit and Switch to Cash Fund - only applicable for unit-linked savings products.

Claim type

Specifies the type of cover that this coverage provides. The available claim types are: Death, Disability, Job Loss, Medical, and Maturity.

The description of these Claim Types can be set for each language using Support Files > Codes > Codes Records > CLAIM_TYPE.

Valid death causes

Death benefits may not be paid for some causes of death. The cause of death is entered when the death claim is registered. If the cause of death is not set here, the death benefit will not be paid.

Cause of Death Codes are defined using: Support Files > Codes > Codes Records > DEATH_CODE.

For each cause of Death, a waiting period (in months) may be set. Death benefit will not be paid on claims registered during this period (which starts from the Policy Issue Date).

Claim Benefit Calculation Code

This field identifies how the claim amount will be calculated. A drop-down list shows the available options. These options are fixed. If a new calculation is required, it will need to be added by FIT.

Claim Benefit Factor Table

If the selected Claim Benefit Calculation Code requires a rate table, this field identifies which rate table to use.

Rate tables are defined using:

Support Files > Rates > Rate Structure

Support Files > Rates > Rate Entry

Claim Benefit Factor

If the selected Claim Benefit Calculation Code requires a fixed rate, enter the rate here.

Claim Benefit Maximum

Some of the Claim Benefit Calculation Codes allow a maximum claim amount to be set. Enter the maximum claim amount here if applicable.

Deduct outstanding premiums from claim amount

Set this check box to make the system deduct any unpaid premiums from the claim amount.

Waiting period (months), can be overridden by claim type

A waiting period (in months) may be set here. Benefits will not be paid on claims registered during this period (which starts from the Policy Issue Date).

For Death claims, any waiting period set for the cause of death overrides the waiting period set here.

Postpone claim benefit payment (months)

Delay payment of claim by this many months.

When create an event for postponed benefit (days before benefit due)

Number of days before a delayed claim is to be paid that the document trigger is created.

Claims are rider-specific

If ticked, the process rider claim button will allow claims on this coverage.

If this check box is ticked, it allows On settlement also settle riders and On settlement terminate riders to be entered.

On settlement, settle plans for all lives

If ticked, all coverages will be settled on claim.

If unticked, coverages with the same life assured will be settled. This enables a policy where there are multiple lives assured to remain active for the remaining lives on the death of one life assured.

On settlement also settle riders

Select any other riders that will be included in this claim.

On settlement terminate riders

Select any other riders that will be terminated by this claim.

Claim Notified status

The plan will be set to this status when the claim is registered.

Claim Settled status

The plan will be set to this status when the claim is settled.

Annuity payment on settlement

If ticked, when processing a death claim on a rider, the benefit will be paid as an annuity. The parameters for the annuity payment are set at Product Workbench > Annuities.

Total claim amount limited to base sum assured

If Annuity payment on settlement is selected, this check box will limit the total claim amount to the base sum assured.

Save button

Saves changes and returns to the previous screen.

Return hyperlink

Return to the previous screen without saving changes.
