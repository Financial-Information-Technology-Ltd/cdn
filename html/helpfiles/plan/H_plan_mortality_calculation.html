Plan Mortality Calculation

Parameters for calculating mortality (life insurance cover) premium where it is paid by cancelling units or reducing deposit administration savings value.

Label

Description

Plan Code

Plan Code display only

Plan Name

Plan Name display only

Age calculation code

Select the age calculation code from the dropdown list.

Options are:

         0 - Issue Age + Policy duration

         1 - Issue Age + Truncated policy duration

         2 - Age at the effective date

This enables a higher premium to be charged as the life assured gets older.

Per Mil Rating

Enter the fixed per mille rating, if any. This is added to the mortality rate obtained.

Age Rating

Enter the fixed age rating, if any. This is added to equivalent age before obtaining the mortality rate.

Minimum Qx

Enter the minimum mortality rate (Qx), if any.

Qx Divisor

Enter the Qx (mortality rate) divisor to be used.

The stored mortality rate is divided by this number - it is a legacy of Cobol systems which could not store decimals, so would store rates as an integer to be divided by a fixed factor at run time.

Mortality Loading

Enter the fixed mortality loading percentage to be used, if any.

Std Table Load

Select the mortality loading factor table (if any) from the dropdown list.

The list is comprised of rate tables defined at Support > Rates > Scheme Rates.

If blank, the mortality rate table identified below (Mortality Rates Table) will be used without rating factors from a table.

Restrict charge deduction

Tick if mortality charge should be processed (paid) but not actually deducted from units.

Sum at risk code

Mortality Premium = Sum at Risk * rate/1000.

Select the sum at risk calculation from the dropdown list.

Options are:

         0 - Face Amount

         1 - Face Amount-Bid value of units

         2 - Face Amount-Bid value of units (can be negative)

         3 - Future premiums on other coverages (for waiver)

         4 - Sum Assured - Reserve Value

         7 - Face Amount-Bid value of units on base coverage

Deduct Sum at risk of riders

When calculating the coverages sum at risk, select riders whose sums at risk should be deducted from the dropdown list.

This is used in evaluating the reinsured sum at risk for related risks, such as death and critical illness, to avoid double charging.

e.g. Death Sum Assured 1,500,000.

Critical Illness Sum Assured 1,000,000.

The company will pay out 1,000,000 on diagnosis of critical illness, and a further 500,000 on subsequent death. They are really the same risk, so the reinsured sum at risk for death is 500,000.

Add Sum at risk of riders

When calculating the coverages sum at risk, select riders whose sums at risk should be added from the dropdown list.

Again used in evaluating the reinsured sum at risk for related risks, such as waiver of premium payment on death.

e.g. the future waived premiums should be included as part of the death sum at risk, if paid in addition to a lump sum death benefit.

Include this rider when calculating policy sum at risk (for riders only)

Tick if this riders sum at risk should be included when calculating the policy sum at risk as part of evaluating a clients accumulated sum at risk (his/her total sum at risk across all policies, used in reinsurance premium calculation and underwriting).

Mortality Rates Table

Select the mortality rate table from the dropdown list.

The list is comprised of rate tables defined at Support > Rates > Scheme Rates.

Modal Payment Calculation

Select the method for calculating a modal mortality rate (qM) from the dropdown list. Mortality rates are stored on tables as annual values (qA)

Options are:

         0 - Standard: qM = qA / No of Mortality deductions in a year

         1 - qM = qA / (No of Mortality deductions in a year - qA)

Mortality Details

Set the mortality premium calculation rules for each life insured basis on which a policy can be sold.

No. of Lives

Select the life insured basis (number of lives) from the dropdown list.

Options are:

         S - Single Life

         J - Joint Life (First Death)

         L - Joint Life (Last Survivor)

         V - Multiple Lives

Minimum Charge

Enter the minimum mortality charge, if any.

If a small sum assured and/or a low mortality rate results in a negligible premium, the company may wish to impose a minimum to cover overheads.

Flat Charge

Enter the flat (fixed) mortality charge, if any.

Risk Calc Code

Select the risk calculation code from the dropdown list.

Options are

         0 Standard (i.e. single life Qx read from mortality rate table)

         1 - Single Prem BRLUX

         2 - 1 - (1 - Qx) ** 12

         3 - Qxt + Qyt - (Qxt * Qyt)

         4 - (Pxt * Qxt * (1-Pyt))… B&W Special

         5 - Qxt + Qyt

         6 - % of Qxt of Oldest (no loads)

         7 - Q / (12 - Q)

Num Rating

Enter the numerical rating if related to the number of lives assured. Will be multiplied by the mortality rate.

e.g. 1.3 will give a 30% loading.

JLLS Var-A

Used in conjunction with Risk Calc Code 4 above (B&W Special).

Joint Life Last Survivor calculation, variable A.

JLLS Var-B

Used in conjunction with Risk Calc Code 4 above (B&W Special).

Joint Life Last Survivor calculation, variable B.
