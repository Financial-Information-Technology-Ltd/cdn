Product Statuses

Select the default policy statuses which will be used when a policys status is set automatically as a result of system processing.

These plan-level defaults override any corresponding system level defaults set at Administration > Configuration > Statuses, enabling different default statuses by product type.

Initial status

Select the default initial status on policy creation from the dropdown list.

The list is comprised of statuses defined of type pending, configured at Support > Codes > Status Codes

Ready to issue

Select the default ready to issue status on policy creation from the dropdown list. This status will be used for new applications with no underwriting requirements set.

The list is comprised of statuses defined as type pending, configured at Support > Codes > Status Codes

Incomplete details

Select the default incomplete details status on policy creation from the dropdown list. This status will be used for new applications with mandatory fields not completed.

The list is comprised of statuses defined as type pending, configured at Support > Codes > Status Codes

Passed to underwriting

Select the default passed to underwriting status on policy creation from the dropdown list. This status will be used for applications with underwriting requirements set (e.g. Sum Assured above underwriting limit).

The list is comprised of statuses defined as type pending, configured at Support > Codes > Status Codes

Ready for underwriting

Select the default ready for underwriting status from the dropdown list. This status will be used for policies whose underwriting requirements have been cleared. Often used to trigger workflow.

The list is comprised of statuses defined as type pending, configured at Support > Codes > Status Codes

Terminated

Select the default terminated status from the dropdown list.

The list is comprised of statuses defined as type Terminated, configured at Support > Codes > Status Codes

Declined

Select the default declined status from the dropdown list. This status will be used for policies whose underwriting requirements have been rejected.

The list is comprised of statuses defined as type Terminated, configured at Support > Codes > Status Codes
