Plan Paid Up

Parameters for paid-up processing when premiums stop but the policy remains on the books. This is applicable to savings contracts.

Label

Description

Paidup Type

Select the type of paid up processing applicable from the dropdown list.

Options are:

         Y - Allowed

         T - Change status to terminated (for coverages)

         N - Not allowed

Sum Assured recalculation code

For endowment savings policies, where the sum assured is paid out at maturity, select the method for recalculation of sum assured on status change to paid-up from the dropdown list.

Options are:

         1 - Sum Assured MINUS (less Paid-to date and in force months/term in months)*(less Paid-to date and in force months/term in years)* Sum Assured)

         2 - Premiums Paid * Sum Assured percent

         3 - Premiums Paid * Rate

         4 - Sum assure * (months already paid / total payment months)

Sum Assured recalculation rate table

For sum assured recalculation code 3 above, select the rate table from the dropdown list.

The list is comprised of rate tables defined at Support > Rates > Scheme Rates.

Table for PUP penalty rate

For sum assured recalculation code 3 above, select the rate table from the dropdown list.

The list is comprised of rate tables defined at Support > Rates > Scheme Rates.

PUP penalty minimum basis

Select the minimum paid-up penalty basis from the dropdown list if applicable.

Options are:

1 - Amount of outstanding Entrance Fee. This is applicable for regular premium unit-linked products where an entrance fee is charged (a fee that is taken on a regular basis at the start of the policys life). If the policy goes paid-up before the end of the entrance fee charging term, the remaining entrance fee is charged.

PUP penalty minimum factor

Paid Up Penalty minimum factor used in conjunction with PUP penalty minimum basis above (i.e. factor to apply to outstanding entrance fee)

Status when policy goes paid-up automatically

Select status to be used when a policy goes paid-up automatically from the dropdown list. The list is comprised of statuses defined as being of status type paid up at Support > Codes > Status Codes.

Minimum policy value when paid-up is allowed

For unit-linked savings policies, enter the minimum policy value for which a policy can go paid up. For policy values below this the policy will lapse and any value will be taken as a charge.

Months after issue before paid-up is allowed

Enter the number of months after policy issue that must pass before a policy can be made paid up. Before this, the policy will lapse and any value will be taken as a charge.

Reinstatement requires catch-up premium in suspense

Tick if it is necessary to pay all missed premiums in order to reinstate a policy from paid-up status.

Do not reinstate

Tick if reinstatement is not permitted

Reinstatement requires underwriting

Tick if underwriting is applied to reinstatement. i.e. in order to maintain life cover, the same medical checks as at new business are required

If this option is selected then you must also select the Schedule Reinstatement on the Reinstatement screen.

Charges taken when policy is in paid-up status

Tick the charges which are still applied for policies in a paid-up status.

Options are

         Administration Fee

         Policy Establishment Fee

         Loyalty Bonus

         Forward Ad-Hoc Unit Withdrawal

         Loyalty Bonus Processing

         Management Charge

         Maintenance Charge

         Regular Income Payment

         Unit Deallocation for Premium (mortality charge)

         Initial Unit Charge

         Risk Premium Deduction

         Morbidity Charge

         Intermediary Admin Fee
