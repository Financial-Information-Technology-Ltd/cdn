Plan Lapse

Lapse Parameters how policies behave upon non-payment of premiums due. Typically used for risk coverages so that life insurance cover lapses (ends) if the policyholder stops paying premium. Multiple warnings can be automatically sent before cover actually lapses.

Label

Description

Lapse allowed

Tick this box if lapses are allowed for policies of this plan type.

Lapse Free Period (months)

Number of months after policy issue for which a policy cannot lapse (even if premiums are late)

Days before automatic premium loan is created

Automatic premium loans can be used instead of lapsing cover.

Enter the number of days after the premium due date that a loan is automatically created with which the policys premium will be paid.

Parameters relating to the automatic premium loan itself are set on screen product workbench > loans.

Days of grace before lapsing

Number of days a premium is overdue before the cover is lapsed. A claim made during this period will be honoured, but the overdue premium will be deducted from any claim amount paid.

On issue, a LA (Lapse) future record is created for the coverages paid-to date (next premium due date) PLUS this number of days. Every time a premium is paid, and the coverages paid-to date moves forward, the LA future record is moved forward also. If LIFEfit reaches this date, and any preceding lapse warnings have been processed, batch job LF.LAPSE.POLICY.BATCH will automatically lapse the coverage.

Please note that if there is any money in suspense on a policy, it will not lapse. It is a good idea to check the batch report (LF211) for information about lapses processed, and lapses that could not be processed (e.g. because there is money in suspense, or there are unprocessed lapse warning triggers).

Lapse Notice section

LIFEfit can automatically send letters (or emails) to a policyholder, reminding them to pay their overdue premium in order to maintain life insurance cover. If premium is not paid within the days of grace, cover will lapse and claims will not be paid.

Days after non-payment

Number of days a premium is overdue before lapse warning letters (or emails) are triggered. Multiple lapse warnings are possible by making multiple entries here.

On policy issue, a LN (Lapse Notice) future record is created for the coverages paid-to date (next premium due date) PLUS this number of days. Every time a premium is paid, and the coverages paid-to date moves forward, the LN future record is moved forward also. If LIFEfit reaches this date (i.e. premium has not been received yet), batch job LFS.LAPSE.REMINDERS followed by LFSDO.IDOCN will automatically trigger a reminder letter or email.

If there are multiple entries here, processing the LN for a warning results in the creation of the LN for the next warning (if there is one).

It is a good idea to check the batch report (LF212) for information about lapses warnings processed.

Documentation Event

Document trigger event to be associated with the warning. If this is left blank, the default event LAPSE_LETTER is used.

Documents are configured at Support >Data > Standard Letters, and associated with trigger events at Admin > Configuration > Documentation (and a Product Workbench > Documentation if product-specific).

Please note the document event name is stored on the LN future record, set at policy issue. If this plan parameter is updated it will only apply to policies issue from that point on. If you need the change to work for policies which are already active, please contact FIT for a data fix.

Make policy paid-up instead of lapsing if paid-up is allowed (see paid-up parameters)

Tick if policy is to be made paid-up instead of lapsed.

For savings policies that go paid-up, the value is crystallised and still belongs to the policyholder. If they lapse, the policyholder loses their money and it is credited to the company.

Leave coverage active instead of lapsing (if it is base or base is left active)

Tick if the coverage is to remain active, even if the base coverage lapses (the default behaviour of LIFEfit is to lapse all riders if the base coverage lapses).

Lapse value

Select the lapse value of a policy from the dropdown list (used for savings products).

Options are

       Total value of units and deposit balance

       Surrender value (which may be subject to a penalty see screen Product Workbench > Surrenders)

Months after issue before lapse can have value

Number of months after policy issue before lapse can have value

What should happen to funded coverage when policy does not have enough funds to pay charges

If a rider risk benefit is paid from the unit-linked savings value of the base coverage, select what should happen if it looks like the units will run out (or have run out)

Options are

       Send a warning when below minimum

       Send a warning when below minimum and lapse when no units left

Lapse Status when policy lapses automatically

Select the status code to be used for lapse from the dropdown list of status codes defined as being of status type lapsed (at Support > Codes > Status Codes)

No commission clawback on lapse

Stops the commission from being clawed back when the policy lapses

Write off Loan balance on lapse

If this flag is ticked then the loan balance will be written off when the policy lapses
