Data Import

Agent

The Import Agents process allows Agent Code details to be imported from a CSV file. This parameter holds the folder and filename of the CSV file.

The import program is LF.IMPORT.AGENTS.xxx. (where xxx is the LIFEfit customers suffix). It is run from a Batch.

Note: When you create a new Batch code, you cannot run it until you add the Batch Code to your list of allowed batches. This is done using Administration > Security/Processes > Operator Groups, Allowed Batches.

Image

Storage

Location at which DocStore (LIFEfit Document Management System) files are stored.

Import

Location from which manually created items are imported to DocStore by batch program LF.LOAD.IMAGES.

Files should be saved in this location with the appropriate prefix to their filename in order to be attached to the required policy, client or agent in DocStore:

       Policy-related items: prefix POL<policynumber>--

       Client-related items: prefix CLI<clientnumber>--

       Agent-related items: prefix AGT<agentnumber>--

e.g. POLAP10000032Valuation query.pdf will be attached to policy AP10000032.

Load

Location from which .inf files, XML format index files generated by Document Server, are read to load documents to DocStore by batch program LF.IMPORT.IMAGES or the equivalent real time service.

This corresponds to the \info sub-folder of the done folder set for the Document server spool file.

Download DocStore Document

Location to which DocStore documents are downloaded used for bulk download of documents for investment trusts (all documents for policies owned by the trust within a specific date range).
