Policy Accounting

This is screen is used to manually create a policy accounting transaction for a policy. The transaction is a record in the POLACCTRN file, and can be seen from the "Policy Activity" hyperlink on the policy workbench.

Label

Description

Policy No (display only)

Policy Number

Name (display only)

Policy owner's name

Coverage (display only)

Confirms that the transaction is being created for the base coverage (coverage zero)

Trans code

Select the type of transaction you are creating from a drop down list.

The list comes from CODES record PAYMENT_TYPES

Depending on the type of transaction, LIFEfit sets the "sign" (i.e. credit or debit) for subsequent fields cash, premium and suspense.

Currency (display only)

Policy currency - this will be used as the currency for the transaction

Cash

Amount to post to cash

n.b. if you are running multiple cash books, you get an extra drop down box of cash account pseudos so you can define which cash account is to be used This parameter is set at Administration > Configuration > System "Multiple Cash Books"

Premium

Amount to post to premium

n.b . not used for first premium payment. This is done by crediting ,money to suspense, then changing the policy's status to active. The suspense > premium transaction is automatically created by the status change event

Eff Date

Effective date for the transaction.

This defaults to the current LIFEfit system date, and can be changed

Loan

Amount to post to policy loan.

Policy loans are normally used for with-profits policies. Policyholders either want to realise some money, or use value in the policy to pay the premium. Unit-linked polices tend to offer partial surrenders if policyholders want to release funds early, although policy loans are sometimes used instead, as a way of avoiding penalty charges, if allowed within the product rules.

A loan balance is maintained on the policy and can be seen on the policy workbench. Any outstanding loan is netted off at surrender or maturity.

Suspense

Amount to post to policy suspense

Batch cash (display only)

If using the batch cash total function, this shows the current amount posted to cash during this batch by this user.

If multiple cash books are being used, the value shown is the amount for the cash book being used in this transaction.

Batch cash total functionality is set at Administration > Configuration > Site Parameters, "Open/Close batches". If turned on, users can open and close their batch at Premium/Payment > Open/Close Batches.

Policy suspense (display only)

Current balance in suspense on the policy

Annualised premium (display only)

Policy annualised premium amount. Includes all coverage premiums

Total modal premium (display only)

Policy modal annualised premium amount. Includes all coverage premiums.

Pseudo

Allows you to make postings to other accounts. By clicking add, you can add as many extra accounts as you like.

Any pseudo can be selected, unless it is in pseudo group "Restricted Policy Accounting" (defined at Support > Accounting > Pseudo Codes). Typically pseudos related to fund accounting will be blocked.

Account no. (display only)

The LIFEfit general ledger account number

Amount

Amount to post to this account

All transactions must be balanced (i.e. sum of credits EQUALS sum of debits)

Agent No.

Agent number

Description

Free format text description

Reason

Direct debit reversal reason (from CODES record REVREASON)

Chq No

Cheque number

TT Ref

Telegraphic Transfer reference
