Unit Movement Entry

Shows the detail of a unit movement transaction

Label

Description

Policy

Policy number

Cvg

Policy coverage number

Created

Server date when the transaction was created

Time

Server time when the transaction was created

Reason

Reason for the transaction.

From CODES record UNIT_REASON.

Unit Date

Unit price date used in the transaction

Operator

User who generated the transaction. These transactions are usually created in batch at end of day.

Transaction No

Transaction number

Reversal No

Reversal transaction number

For transactions which have subsequently been reversed (e.g. reversal of unit allocation following a premium reversal), this is the transaction number of the associated reversal

If blank, the transaction has not been reversed

Trans date

System effective date when the transaction was processed

Description

Transaction description

Source

If the transaction was created as an adjustment, UAD is shown here

Fund

Code and name of unit-linked investment fund

Ccy

Investment fund currency (which can be different from policy currency)

Units moved

Number of units created / cancelled

A positive number indicates units were created (e.g. following a premium payment)

A negative number indicates units were cancelled (e.g. to pay a mortality premium)

Unit price

Unit price used in the transaction

Value

Value of the unit movement, in fund currency

i.e. (Number of Units moved * Unit Price)

<Return> Hyperlink

Returns you to the list of unit movements
