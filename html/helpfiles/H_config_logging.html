Tables with changes logged

Select the files whose changes should be audit logged. Any changes are logged date, time, user, field changed, value before change, value after change.

For details of the contents of each file, please see Administration > Configuration > Data dictionary. The logs can be reported on at Administration > Audit Logs

Data logged for XML export

Where LIFEfit and eSurance are run on separate databases, and data is passed between them (typically daily) by XML file, this list defines which tables data is exported from LIFEfit to eSurance.

Each time a record is updated in LIFEfit, its key is written to a log file. A batch process reads the log file and includes that records contents in an XML format export file, to be imported to eSurance.
