Unit Movements

Lists policy unit movements

Click on an individual transaction for further details

The listed is sorted in reverse chronological order by default. You can change the sort order by clicking on a column heading. Clicking again reverses the sort order

Label

Description

Eff date

Effective date of the unit movement

Reason

Reason for the unit movement, from CODES record UNIT_REASON

Cov.

Policy coverage on which the unit movement took place. Units can be held at coverage level, although some companies choose to hold all units on the base coverage.

Price Date

Unit price date used in the unit transaction

Fund

Investment fund (this list contains one row for each fund involved in a transaction)

Units Moved

Number of units created / cancelled

A positive number indicates units were created

A negative number indicates units were cancelled

Unit Price

Unit price used in the transaction

Trans.No

Transaction number.

This number is shared with the associated unit accounting transaction

Rv. No

For transactions which have subsequently been reversed (e.g. reversal of unit allocation following a premium reversal), this is the transaction number of the associated reversal

Source

If the transaction was created as an adjustment, UAD is shown here
