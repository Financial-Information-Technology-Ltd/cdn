Premium Receipt History

This screen shows the payment receipts for a policy, both those which have reconciled, and those which are waiting to reconcile (uncleared cheques)

Payments can be created

-        online (screen Premium/Payments > Online Prem collection)

-        by CSV import (screen Premium/Payments > Import Bank Transactions)

-        by WebService call

Label

Description

Policy

Policy Number.

Batch

Batch counter for import.

Serial No

Serial number for import.

Created Operator

User who created the payment.

Created Date

Server date when payment created.

Created Time

Server time when payment created.

Amount

Amount of Payment.

Type

Payment type (e.g. cash, cheque, electronic funds transfer).

Cheque No

Cheque number.

Receipt No

Receipt number

n.b. it is not possible to create a payment with a duplicate receipt number.

Currency

Currency of the payment.

Bank

Bank sort code.

Reconciled Cleared Date

Date payment cleared.

Uncleared payments (e.g. cheques) will show a blank here until reconciled.

Reconciled Effective Date

Effective date of payment.

Payments can be backdated.

Reconciled User

User who reconciled the payment.

Payments can be reconciled automatically on input if the days to clear for the payment type is zero typically this is used for cash payments.

Payments can be reconciled online, at screen Premium/Payments > Unreconciled transactions. This is often used for payments which could not initially be matched to a policy, once the correct policy is identified. It is also used if a cheque clears before its expected clearing date.

Payments can be reconciled in batch. This is typically the case for cheque payments, n working days after they are received (n is defined at Support > Codes > Cash Payment Types).

Reconciled Date

Server date when payment reconciled.

Reconciled Time

Server time when payment reconciled.

Reconciled How

Indicates the method of payment reconciliation

-        A: Automatic batch reconciliation (Batch program LF.RECONCILE.BANK.TRANSACTIONS clears cheques on their clearing due date).

-        C: Cancelled (on Premium/Payment > Unreconciled Transactions, typically done if a cheque bounces before its cleared date)

-        M: Manual (Premium/Payment > Unreconciled Transactions)

-        O: Online (Premium/Payment > Online Premium Collection, or Premium/Payment > Import Bank Transactions)

-        S: by SOAP (WebService)

Rv

Reversed indicator - contains an asterisk (*) if payment has subsequently been reversed.
