Commission Type Banding Structures

LIFEfit supports commission types which vary by either policy term or sum assured band. This is used, for example, when companies want to pay higher levels of commission for longer policy terms, or for higher sums assured.

The plan is set to use a dummy commission type with a ] (wildcard character) as its last character e.g. AB]. When a policy is issued, the wildcard is substituted by the modifier applicable to its term or sum assured as appropriate e.g. AB7. The policy uses the real commission type rules as appropriate e.g. AB7.

Label

Description

Commission Type

Three character commission type code, ending in ] (wildcard character).

Sometimes referred to as the dummy commission type.

Modifier Type

Character to be substituted for the wildcard on policy issue

You need to create a commission type for each possibility in this column, using the first two characters of this commission type as the prefix.

LIFEfit will not allow you to enter a value for which the matching commission type (once the ] wildcard is substituted) does not already exist.

Modifier Term

Used for commission type varying by policy term (in years).

At policy issue, a policys commission type will be set by concatenating (joining) the first two characters of the dummy commission type, and substituting the modifier type for the ] (wildcard).

Modifier Amount

Used for commission type varying by sum assured band. This value sets the upper band (i.e. the first row will be used for sums assured from zero up to this value.

At policy issue, a policys commission type will be set by concatenating (joining) the first two characters of the dummy commission type, and substituting the modifier type for the ] (wildcard).
