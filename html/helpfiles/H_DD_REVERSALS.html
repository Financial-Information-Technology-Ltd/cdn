Direct Debit Reversals

This screen is used for direct debit reversals. A batch process will automatically pick the reversal records up and process them.

Label

Description

Batch No

Batch no is used for identifying reversal records. It can also be used for adding policies to an existing reversal record or used to create a new reversal record by entering a new batch no.

Submitted for batch

If Submitted for batch is checked in then the batch process will process the request for this DD reversal.

Effective Date

This is the effective date for the DD change to take place. It will usually be the current system date.

Batch Total

Total reversal must tally up to Batch Total.

Policy No

This is the policy no the user is requesting the DD reversal for.

Payer

The surname of the payer on the policy selected.

Account No

The bank account no of the payer.

Reason

Reason code selected to specify the reason for the DD reversal.

Bank Charges

Any charges that apply will be entered here.

Last Premium

Last premium payment amount.

Total

Total is the sum of all reversal transactions on the screen. It includes any bank charges as well.

Date

This field is a protected field. It displays the policy paid to date.

<Delete>

Deletes the selected transaction.

<Add>

You can request for more then one DD reversal.

<Save>

Upon clicking this, LifeFit will save any changes to the screen.

It will also allow multiple policies to be entered.
