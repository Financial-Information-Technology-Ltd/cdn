General Accounting Journal

This screen enables manual accounting postings to be made. Submitting the screen creates a record in the GENACCTRN (accounting journal) file. It will be posted to the general ledger and copied to journal history file GENACCTRN_POSTED the next time the accounting batch is run.

Please note there is no enquiry screen for this data. It can, however, be reported on in LIFEfit ('Davywriter'), or via ODBC e.g. Query via MS Excel.

Label

Description

Effective date

Enter the effective date for the transaction, or click the calendar and pick a date.

The current LIFEfit system date is offered as a default.

Branch

Select the branch for this transaction. The company code is automatically set, based on this (each branch belongs to only one company).

Currency

Select the currency for this transaction. Any currency that is active (Support > Currency > Currency Codes, "Active") can be used.

Reference

Enter a reference. This can be anything.

Description

Free-format text.

Something must be entered here - it cannot be left blank.

Pseudo

Select the accounting pseudo from the drop down list. All pseudos are available.

Plan

If the pseudo (selected above) is mapped to plan accounts, select from the drop down list of plan codes.

If the pseudo is mapped to fund accounts, this field is skipped.

Fund

If the pseudo (selected above) is mapped to fund accounts, select from the drop down list of fund codes.

If the pseudo is mapped to a plan accounts, this field is skipped.

Account (Display only)

The LIFEfit account number.

o        for fund accounts, this is set at Support > Accounting > Fund Accounts

o        for plan accounts, this is set at Support > Product Workbench, "Accounting"

Amount

Enter the amount

o        negative: Credit

o        positive: Debit

n.b. you must enter a "balanced" transaction, where the sum of credits equals the sum of debits

"Add" button

Click to add more rows

"Submit" button

This saves the transaction in journal file GENACCTRN.

Your user ID will be written on the transaction, as well as the time and date.
