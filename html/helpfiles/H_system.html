Description

Min.Password Length

Minimum password length (validated at password change).

Password should include numbers and letters

Tick if login password must include numbers and letters for additional security (validated at password change).

Password is case-sensitive

Tick if login password is case sensitive for additional security (validated at password change).

Force Password Change after < > days

Number of days after which password change will be forced on login to LIFEfit.

Force eSurance Password Change after < > days

Number of days after which password change will be forced on login to eSurance.

Maximim Login Attempts

Maximum number of failed login attempts, after which the user ID is set to inactive. The user ID may be reactivated by the system administrator.

Lock Web User permanently when maximum login attempts reached

Tick if the eSurance ID should be flagged as inactive on reaching the maximum number of failed login attempts. In this case, it must be reactivated by the system administrator. If not, it will be temporarily deactivated, according to the parameter below.

Timeout when Web user is locked when maximum login attempts reached by lock is not permanent < > minutes

If the parameter above is not ticked, the length of time for which an eSurance ID is inactive after reaching the maximum number of failed login attempts.

Prevent Keylogging

Additional security option for eSurance login

If ticked, the eSurance login procedure requests three, randomly selected characters from the password to be entered (e.g. first, fourth and seventh).

If not ticked, the eSurance login procedure requests full password entry.

Clear sensitive information on end-of policy cycle after < > days

After a policy matures/expires, the number of days after which bank details are deleted.

SSL Client Certificates for ESurance login

Tick if SSL (Secure Sockets Layer) Client Certificates are required for eSurance login for all eSurance users.

SSL Client Certificates for ESurance Agent login

Tick if SSL (Secure Sockets Layer) Client Certificates are required for eSurance login for all agent users only.

Language

Code Page for Data (if system is not set for Unicode)

For units, use coverage issue date

Account for Forward Pricing Report

LIFEfit offers forward pricing functionality, modelling the next working days unit movements assuming no change in unit price. This enables companies to know how much of each investment fund to but or sell, based on known transactions.

This parameter identifies the LIFEfit data account in which the modelling is done. A batch process copies the relevant tables from the live data, and processes unit transactions in this account.

Use valuation prices for forward pricing report

Tick if valuation process should be used for the forward pricing report.

Run forward pricing on system date

Tick if forward pricing should be run on the system date (the default is to run it on the next working day).

Save Forward Price report to Pending Price Table

Tick if the forward pricing is to be saved to the pending price table.

Pending Price Tolerance Percentage

Enter the pending price tolerance percentage.

Skin

Select the LIFEfit skin (colour scheme) from the dropdown list. Additional skins can be created as stylesheets, and saved in the \webapp\<accountname> folder. They will appear in this dropdown list.

Companies often choose different skins for test and live accounts, to make it easy for users to know which system they are connected to.

Skin for ESurance

Select the eSurance skin (colour scheme) from the dropdown list.

Date format to use in PDF documents

       M - Medium (e.g. 17 Mar 1990)

       L - Long (e.g. 17 March 1990)

Date format to use on HTML pages

Select the date format to be used in html pages (LIFEfit/eSurance screens) from the dropdown list. Options are

       S - Short (e.g. 24/12/14)

       M - Medium (e.g. 24 Dec 2014)

       L - Long (e.g. 24 December 2014)

How empty value is represented on LIFEfit

Select the way an empty (null) value is represented in LIFEfit from the dropdown list. Options are:

       Space character

How empty value is represented on Esurance

Select the way an empty (null) value is represented in eSurance from the dropdown list. Options are:

Process to call on policy accounting posting

Select the process (if any) to be called on generating a policy accounting transaction. This enables a real time link between LIFEfit and the corporate GL (general ledger). It supersedes the normal way LIFEfit passes information to the GL, by file transfer in the end of day batch.

Address Expiry Years

Used to set address expiry date on policy issue (issue date + years).

Batch process LF.EXPIRED.ADDRESS checks the client address expiry date against the system effective date. If the address expiry date on or before the system effective date, a list type (document trigger) ADDRESS_EXPIRED is created for the client. If a parameter is passed into the batch process, that many days are taken off the system effective date before the check is made. This allows advance warning of address expiry.
