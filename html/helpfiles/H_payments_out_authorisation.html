Payment Out Authorisation

LIFEfit features a two stage payment out process:

1.       Payments are initiated as part of regular customer servicing activity such as surrenders, death claims and maturities

2.       This screen is used where the business process requires second user authorisation before the actual payment out is made

User groups feature two levels of authorisation limit:

1.       limit for payments that can be made without second user authorisation

2.       amount that can be authorised, for a payment initiated by another user

These limits are set on the user group profile (Admin > Security > Operator Groups), and can be overridden for individual users (Admin > Security > Security). Different limits are set for different currencies. Typically, the more senior the user, the higher the authorisation limit. All authorisation is fully audit-trailed.

Regular payments out (e.g. commission, annuity payments and regular income payments) do not require individual authorisation for each instance. In addition, if no limits are set on either the user group or individual user, this screen is bypassed and payments are made without second user authorisation.

On submitting this screen, the appropriate payment trigger future records are created (DC_xx for direct credit, DC_TT for Telegraphic Transfer and CHQ for cheque payments). These are processed in batch.

The history of payments out for a policy can be seen from a hyperlink on the policy workbench (enquiry) screen.

Payments Out heading

%

Used where a payment is to be split across multiple recipients. If you enter less than 100%, additional rows are added until the percentage totals 100.

Type

The method of payment.

This is a drop down list based on the available methods defined at Admin > Configuration > System Date Limits / General, "Payment Out Types "

Currency

The currency of the payment out.

Available currencies are defined by payment out type at Administration > Configuration -> system flags screen.

If the payment out is in a different currency from the policy, it will require an exchange rate within the backdate limit in order to process. Exchange rates are set at Support > Currency > Currency Rates. The backdate limit is set at Administration > Configuration > System Date Limits / General, " Exchange rate back date limit".

Charge amount

Amount to deduct for bank charges. Will be posted to accounting pseudo BANK_CHARGE

Payee No

LIFEfit client number of the payee - only required for electronic payments (TT and direct credit)

Payee Name

For cheque payments, this can be entered

For electronic payments this is display only, taken from the LIFEfit client

Bank no

The bank account to which the payment is being made - only required for electronic payments.

Edit

Allows you to amend an existing bank account record to be used for the payment (only required for electronic payments)

Create

Allows you to create a new bank account record to be used for the payment (only required for electronic payments)

Additional instructions

Free format text. Will be used in the electronic payment file (TT or direct credit)

Adjustments already taken

Any adjustments (e.g. bank charges, ex-gratia payments) already made to the gross payment

Adjustments
