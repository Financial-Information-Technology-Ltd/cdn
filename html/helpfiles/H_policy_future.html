Policy Future

Shows the dated list of event triggers for the next event of each type for the policy e.g. billing, charging, lapsing.

A combination of the product rules and data entered at new business results in a set of future event triggers being created on policy activation.

Future events are processed automatically in batch when their processing date is reached. As a general rule, there is a 1:1 relationship between a future event type and a batch process.

Future events are processed only if all supporting information, such as unit prices and exchange rates, is available to enable them to process correctly. On processing, the next record is created for the next processing date, according to policy and product rules e.g. on a monthly-billed policy, the next billing trigger is set up for one month later.
