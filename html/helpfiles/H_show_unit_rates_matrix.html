Unit Rate Matrix

Rate tables are used in many places in LIFEfit, for example premium rate tables, unit-linked allocation rate tables and surrender penalty rate tables.

This screen is used to view, create and edit matrices of rates (i.e. where there are multiple columns in the table). Matrix rates are often used where a rate value varies according to both policy term and expired duration.

LIFEfit offers the ability to import and export matrices of rates from and to CSV format this can be very useful when you are working with spreadsheet product models, as their rates can be loaded without re-keying and risk of error.

The top section of the screen is display only, and shows the rate table key as selected (or entered) on the previous screen. It shows the X (column) and Y (row) axes of the rate table.

On entering the screen for the first time for a matrix rate table, you will be prompted to set the X and Y axis values (column and row headings).

Tip: If you have multiple tables to create in the same X/Y format, you do not need to set the axes for each individually, you can import them from CSV

Label

Description

<Edit> button

Opens the rates as text boxes, enabling you to update rate values

<Add> button

Appears when you are editing rates, and enables you to add rows to the table

<Save> button

Appears when you are editing rates, and enables you to save changes.

On saving, you are returned to the rate table selection screen.

<Edit Axis> button

Allows you to edit the X and Y axes of the table (column and row headings)

<Import> button

Enables you to import rates from a CSV file

Tip: when importing a new version of existing rates, first export the existing rates to ensure you get the right row/column layout in your CSV file

<Export> button

Enables you to export rates to a CSV file

<Return> hyperlink

Returns you to the rate table selection screen.
