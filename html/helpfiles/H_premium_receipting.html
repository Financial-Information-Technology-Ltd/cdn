Premium Receipting - Cash Payments

Select the policy for which you are registering a cash or cheque payment.

Pressing "Continue" extends a simplified accounting screen. This screen was originally designed for use in a front desk "cash receipting" environment, but can also be used in the back office.

Label

Description

Policy No

Policy Number

Owner Surname

Enter the first four (or more) characters of the policy owner's surname. This helps ensure you do not accidentally credit money against the wrong policy.

Plan code

The policy's base (coverage 0) plan code and plan name

Payment Type

Pick from the drop down list of payment types that are defined on the system, set in CODES record CASH_PAY_TYPE

Currency

Select from the drop down list of currencies available on the system. The policy currency is offered as a default

Amount

The amount of money being receipted

Cash PSEUDO

If you are running a single cash book, LIFEfit uses pseudo CASH.

If you are running multiple cash books, select from a drop down box of cash account pseudos to

define which cash account is to be used

This parameter is set at Administration > Configuration > System "Multiple Cash Books"

Cash account pseudos are defined by pseudo group "Cash Accounts" at Support > Accounting > Pseudo Codes.

Cleared Date

This is "n" working days ahead of the system date, where "n" is the related value from CODES record CASH_PAY_TYPE. Non-working days (weekends and holidays) are "skipped".

This enables cheques to be registered on LIFEfit, and automatically reconciled with the policy on a date in the future.

Different types of cheque (i.e. drawn on different banks in different countries) take different lengths of time to clear. If a cheque bounces, the payment can be stopped (by clearing this field) before it is credited to the policy.

When they reach their "cleared date", cheque payments are credited to their policy by batch job LF.RECONCILE.BANK.TRANSACTIONS.

Cheque No

Cheque number - for information and subsequent enquiry.
