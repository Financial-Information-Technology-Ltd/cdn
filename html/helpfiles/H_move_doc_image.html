DocStore Item

DocStore is LIFEfits document management system, used to store incoming and outgoing correspondence including LIFEfit-generated policy documents, letters and e-mails. It can also be used to store pictures, sound (e.g. recordings of client telephone calls) and video.

Enter the correct details of the DocStore item selected on the previous screen. This is typically used when a document has been attached to the wrong policy, or needs to be reclassified as sensitive so not all users are able to view it. You cannot hide a document by removing all links to other entities (policy, agent, client etc.) LIFEfit insists that at least one link remains.

A DocStore items number can be seen, it the bottom left of the browser window, when you hover the mouse pointer over its hyperlink.

Label

Description

Item ID

DocStore ID, as entered on the previous screen. Display only.

File Name

File name. Display only.

Source

Source of file. Display only.

       B - Bulk-Imported

       E - Uploaded from eSurance

       I - Imported

       L - Generated by LIFEfit

       U - Uploaded by user

Created By

Operator who uploaded the item. Display only.

Created On

Time and date of the upload. Display only.

Title

Update document title free format text.

Policy Number

Update policy number. Changing this will move the item to a different policy.

Client Number

Update client number. Changing this will move the item to a different client.

Agent Number

Update agent number. Changing this will move the item to a different agent.

Group Number

Update group number. Changing this will move the item to a different group scheme.

Group Member Number

Update group member number. Changing this will move the item to a different group scheme member.

Workflow Task Number

Update workflow task number. Changing this will move the item to a different workflow item.

Document Category

Update document category select from the dropdown list.

Changing this will recategorise the document. It is possible to control access to categories of document by user group (e.g. new business clerks may not be given access to medical documents, these may be restricted to underwriters only).

Sensitivity

Update document sensitivity. Integer 1 to 10 (with 10 being most sensitive).

Document access can be restricted to specific user groups according to their sensitivity e.g. only supervisors are allowed to see documents with sensitivity 7 or higher.
