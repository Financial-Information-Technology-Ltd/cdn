Unit Accounting Entry

Shows details of a unit accounting transaction

Label

Description

Date

Server date when the transaction was created

Time

Server time when the transaction was created

Operator

User who generated the transaction. These transactions are usually created in batch at end of day.

Policy

Policy number for which the transaction was generated

Coverage No

Policy coverage (plan) for which the transaction was generated

Company

Company code of the policy

LIFEfit supports concurrent multi-company operation on the same database, with separation of accounting for reporting

Unit Date

Unit price date used in the transaction

Branch

Branch code of the policy

Transaction No

Transaction number

Trans Date

System effective date when the transaction took place (note necessarily the effective date of the transaction)

Amount Total

Total amount posted to unit creations (or unit cancellations) for the transaction

Reason

Transaction reason (from CODES record UNIT_REASON)

Source

If the transaction was created as an adjustment, UAD is shown here

Genldg Pseudo

Accounting pseudo

Gen Ledger Acct

Account number to which the pseudo is mapped. Mapping pseudos at plan and fund level gives a high level of granularity for analysis

Gen Ledger Amounts

Amount posted to account.

LIFEfit ensures all transactions result in a balanced accounting posting

Ccy

Currency

Funds

Unit-linked fund

Amount tag

Indicates if posting relates to

-        bid-offer spread (BO), the difference between buy and sell price, posted as a profit to the company

-        fluctuation (FLUCT), the difference in unit prices between current system date and date of backdating for backdated transactions, which is a profit if prices have fallen, and a loss if they have risen.
