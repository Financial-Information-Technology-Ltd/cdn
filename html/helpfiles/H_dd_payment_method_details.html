Description

Payment method name (display only)

Account Name

Sort Code for Direct Debit

Account Number for Direct Credit

Bank account number to be used for direct credit instructions

Sort Code for Direct Credit

Bank sort code to be used for direct credit instructions

Reject Limit

Number of times to submit a direct debit instruction before putting it on hold awaiting manual intervention.

If an instruction fails, it will be automatically resubmitted until this limit is reached

Output File

Output (interface) file name

Output File for Direct Credit (if different)

If a separate output (interface) file is required for credit payments, define the file name here

Reference Number

Reference number used in output (interface) file

Output line for each transaction even for the same policy

Where there are debit instructions for multiple premiums for a client on the same day, by default LIFEfit will combine them into a single row in the output file in order to minimise bank processing charges.

Tick this option if you would prefer to have a separate row in the output file for each debit instruction.

Cash Pseudo (if different from default)

Select cash accounting pseudo to be used for transactions using this payment method from the dropdown list.

The list is comprised of accounting pseudos in pseudo group C Cash Accounts.

OS process to run after direct credit file is created

Used to define an external process to be run after creating a direct credit file.

This can be used to call a file encryption process, where the processing bank requires an encrypted file to be transmitted for reasons of security
