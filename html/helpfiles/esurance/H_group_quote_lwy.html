Scheme Setup

This screen allows you to generate a quote for a new Group Life Scheme.

Quote parameters are set up at scheme level, and many can be overridden at subgroup level. This allows a quote to have different rules for different categories of employee.

Once the scheme and subgroup rules are defined, member information can be entered or imported. You can then calculate the premium and produce the quote document

Annual premium

Annual scheme premium (display only)

Renewal period (also known as pro rata period)

Where the scheme is annual, but the first renewal is in less than one year, enter the number of months until renewal here.

Quoted premiums will be pro-rated i.e. n/12 of a full annual premium, where n is the renewal period.

If left blank, a full annual premium will be calculated.

Base Coverage Configuration

This section defines the rules for the group life coverage

Sum Assured Basis

Select basis from dropdown list

-        Multiple of Salary

-        Fixed Amount

Sum Assured Value

If a multiple of salary scheme, select the salary multiple from a dropdown list. Permitted salary multiples are defined in the plan rules (product workbench > sum assured)

If a fixed amount scheme, enter the amount. Subject to maximum defined in the plan rules (product workbench > sum assured)

This value can be overridden at subgroup (employee category) level

Product Configuration

This section defines the additional (rider) benefits applicable to the scheme.

Product

Tick the rider benefits applicable to the scheme

SA Basis

Select sum assured basis from dropdown list

-        Multiple of Salary

-        Fixed Amount

SA Amount

If a fixed amount scheme, enter the amount.

This value can be overridden at subgroup (employee category) level

SA as multiple of salary

If a multiple of salary scheme, select the salary multiple from a dropdown list

This value can be overridden at subgroup (employee category) level

Subgroup Details

Shows the subgroups (employee categories) defined for the scheme. Many scheme level parameters can be overridden at subgroup level
