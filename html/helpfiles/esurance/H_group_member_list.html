Scheme Members

This screen lists scheme members, and allows you to add members. It also allows you to authorise (or cancel) member detail changes if permitted according to your user profile.

Members with changes awaiting second user authorisation are shown at the top of the list.

Pending changes?

Shows Yes if changes have been made which affect the premium (e.g. salary, where sum assured is a salary multiple) , but the supporting premium has not yet been paid.

Authorise Changes

Shows Authorise and Cancel buttons for changes requiring second user (maker/checker) confirmation, where you are permitted to authorise the change.

Schemes can be configured to allow maintenance of member details (including joiners and leavers) by either the agent or the scheme owner. Any change must be authorised by a second user.

e.g. if you are an agent on an agent-maintained scheme, and a change of salary has been made by another user from your organisation, you can authorise or cancel the change. It will automatically update member details in the back office.

<New Member> button

Allows you to add a new member to the scheme. This may require second user authorisation, as described in Authorise Changes above.
