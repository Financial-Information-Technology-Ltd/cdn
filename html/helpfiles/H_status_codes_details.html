Policy Status Codes

Much LIFEfit processing is governed by a policys status.

This screen is used to set the processing rules for policy statuses, and allow companies to map their own rules for policy status flow and processing in LIFEfit. It also allows them to use status codes and descriptions they are comfortable with (i.e. they are not hard-coded into LIFEfit itself)

In addition, it allows the triggers for document generation and workflow creation based on policy status change events to be defined.

Status Type

Select status code type from the dropdown list.

LIFEfit processing is based on status code type; this enables you to use the code and description (above) you are comfortable with.

         L - Lapsed

         MB - Maturity Benefit Paid

         N - Not Taken

         PSUR - Pending Surrender

Remove sensitive information from database on this status

Removal of sensitive information (credit card number) is scheduled on change to this status.

The delay (in days) before data removal takes place is set at Admin > Config > System : Clear sensitive information on end-of policy cycle after 'n' days

Target codes

Use this section to define the status flows that are possible.

When a policy is in this status, these are the options that will be presented when you on the status change screen, in policy maintenance.

LIFEfit maintains some fundamental life insurance rules on status flow between status types which will override the settings here e.g. you cannot process a death claim on a pending policy.

Valid for Group

Defines whether this status is applicable to group master policies, and will be offered as a status change option for them.

Valid for Certificates

Defines whether this status is applicable to group certificates, and will be offered as a status change option for them.

Event generated when group status changed

Document / workflow trigger generated for a group master policy on change to this status

Events

This section defines what document / workflow triggers are created on change to this status. These are associated with documents or workflow tasks at screen administration > configuration > documentation (which can, in turn, define the event as specific to a company or a plan type)

Event Name

Document / workflow trigger event name.

Free format, but will have to match the document event defined at administration > configuration > documentation

Coverage

Tick if the event is generated on rider status change

Original Status Type

Select a status here if the event depends on the status the policy is changing from, as well as the status it is changing to.

e.g. you may wish to trigger different documents if a policy is being activated for the first time (changing from pending status) than if it is being reinstated from lapsed.

<Add> button

Add another document trigger event for this status.

You may have multiple rows here, for example if you want different events for riders than for base coverages, or if there are multiple workflow tasks to be initiated
