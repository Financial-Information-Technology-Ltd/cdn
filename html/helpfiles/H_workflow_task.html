Workflow Task

This screen is used to initiate new workflow tasks, usually as the result of incoming client correspondence.

After saving, the task will appear in the personal "to do" list on the main LIFEfit screen of the user it is assigned to. If not assigned to an individual, it will appear in the secondary user group "to do" list for all members of the task's default user group.

Label

Description

Task No (display only)

Task number - a unique identifier, assigned by LIFEfit.

Task

Select from the drop down list of task codes.

These represent all the processing functions that the company has decided to track, and are defined at Codes > Workflow Task codes

Start Date

Start date for the task - used as a basis for deciding target dates.

Deafulted to the current calendar date. If you are allowed to by your user profiel, yo can change this date (e.g. for setting reminders which will appear in the future, when the start date is reached).

Finish Date

Start Date + "target processing days" (from Workflow Task code dfefinition). This is the date the task will "go amber" in the to do list. It skips non-working days (weekends and holidays)

Due Date

Start Date + "max processing days" (from Workflow Task code dfefinition). This is the date the task will "go red" in the to do list. It skips non-working days (weekends and holidays)

Assigned to

User the task is assigned to - and in whose "to do" list it will appear.

Some users are allowed to reassign tasks to others, according to their user profile. This is a functionality which is typically used by managers and supervisors.

Policy No

Policy to which the task relates.

A search option is available,. allowing you to find a policy by either its application number or external reference (sometimes used to store its "old" policy number on a previous legacy administration platform).

LIFEfit validates that the policy exists. If this field is completed, this workflow task will appear in the list when you click on the "workflow " hyperlink on the policy workbench

The policy owner's name is displayed to the right for information.

Group scheme No

Group Scheme Number

A search option is available,. allowing you to find a scheme by either scheme owner (employer) name or company registraion number.

LIFEfit validates that the scheme exists. If this field is completed, this workflow task will appear in the list when you click on the "workflow " hyperlink on the group scheme workbench.

Client No

Client Number

A search option is available,. allowing you to find a client by surname, Personal ID number or date of birth. Whereas a policyholder will probably know their policy number on LIFEfit, they are unlikely to know their client number.

LIFEfit validates that the client exists. If this field is completed, this workflow task will appear in the list when you click on the "workflow " hyperlink on the client workbench

The client name is displayed to the right for information.

Agent No

Agent Number

A search option is available,. allowing you to find an agent by name or contact name ,

LIFEfit validates that the agent exists. If this field is completed, this workflow task will appear in the list when you click on the "workflow " hyperlink on the agent workbench

The agent name is displayed to the right for information.

Description

Free format text description

Source

Select the source of the request from a drop down list (e.g. letter, fax, email).

This comes from CODES record WORKFLOW_SOURCE. This codes record can be extended, and it is translated on the codes record.

This information is often used for management information reporting (e.g. number of client letters received in a month).

Upstream workflow item

If this workflow item was created automatically by the completion of a previous task in a "chain" of workflow, the previous task's details are shown here.

Downstream workflow items

If this workflow item has automatically created one or more subsequent workflow items on its completion as part of a workflow "chain", details of those tasks are shown here.
