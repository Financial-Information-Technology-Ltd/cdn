Standard Letter

This screen defines the data available to letters that are generated by events in LIFEfit (e.g. policy issue, anniversary, maturity). Letters can also be triggered manually, or as part of a mailshot. In addition, the screen specifies whether the letter is to be physically printed, or sent as an e-mail attachment.

All documentation generated within LIFEfit can be automatically stored in DocStore, the application's DMS (Document Management System) component, and is available from a hyperlink on the enquiry screens.

Label

Description

Merge Code

The name of the standard letter (record in table DOCUMENT_MERGE), as selected or entered on the previous screen.

Description

Free format text description.

Table Name

Name of the LIFEfit table from which data fields are taken for this letter. e.g. if this is a policy-related document, selecting POLICY will here will ensure all fields on the policy table can be output.

If the "type" (below) is "database field", the user will be able to select from a list of all fields on this table.

Template Type

Template type.

Select from the drop down;

- Template - document uses a MS Word document template

- Program - document is generated by its own program (this type of document is rare)

- Not applicable - used when defining data fields to be inherited by another "real" template (see "Extends letters block" below)

Template Name

Name of the MS Word document template to be used (.dot file in Document Server "templates" folder).

Enforce Language Tag (check box)

If checked, this will append an underscore and a two character language code to the template name that will be used.

The language is according to the client's preference, and is set on their client record, though it can be overridden at policy level.

This enables multi-language operation, with the same event generating different letters for different policies (by calling different document templates)

e.g. POLICY_ISSUE_EN for English Language, and POLICY_ISSUE_NL for Dutch language.

There is no need to have separate plan codes to enable multi-language document output.

Language code (e.g. EN or NL) comes from CODES record LANGUAGES.

Include Group No

This is only relevant for Group business. If checked then this document becomes available in DocStore on the Group master policy.

Attach Document to Workflow Task Code

This allows the document to be automatically attached to a Worflow Task. If the system can find an Active Workflow Task with this specified Task Code, the document will be attached to it.

Document Category

LIFEfit has functionality to restrict access to DocStore items. This is achieved by setting a document category and sensitivity on each document.

Document Category allows a LIFEfit document to be categorised according to the user extendable codes record DOCUMENT_CATEGORY. The category of documents a user can access are then defined on the user security profile.

Sensitivity

Once a document has been categorised, the document needs to be given a sensitivity number where number 1 is the lowest.

A LIFEfit user can only access a document from DocStore if the document category set on their security profile matches and the sensitivity number set on their security profile is higher than or equal to the sensitivity number set on the document.

Send By Email (check box)

If checked, the document will be sent as an e-mail attachment in PDF format.

If checked, a subsequent "block" of five fields related to the e-mail's properties is displayed.

If unchecked, the five e-mail fields disappear.

n.b. the e-mail will be sent "from" the address defined in the Document Server module (click on "…" at the send by email setting).

Email Subject

Contains

- e-mail subject line (free format text), OR

- template (.txt file) to use for e-mail subject line (omit .txt extension)

Use template (check box)

If checked, the "Email Subject" above is used to define the text file template.

If using a template, language tags can be used (see field "Enforce language tag" above) so different e-mails can be generated according to client language preference.

For example, template "subject_EN.txt" would be used for English language, and "subject_NL.txt" would be used for Dutch language.

E-mail templates can contain fields defined as labels in the merge, included in the .txt template file as "{ }" delimited variables, just as in .dot MS Word templates. This enables a personalised look and feel to the e-mail.

n.b. omit the template file extension (.txt). This will be added by LIFEfit, after the language tag has been added (if appropriate).

Extends letters block

Inherited template

This enables documents to "inherit" data from existing documents. It saves having to define each data field individually for each document.

It also enables similar data to be output for different entities e.g. you may wish to have client data (name, address, date of birth) for multiple clients with a role on a policy. In this case, the client data is defined once only, in its own "standard letter" template, and "called" here once for each client, using the data field with the client number as the "Key Value" (below).

Key Type

Select from the drop down

- Database field - key value for the inherited template is a field on the record (from field "Table Name") above

- Process - key value for the inherited template is defined in a program.

Key Value

- Database field. In this case, the "search" button will offer a popup window of all fields on the table, in alphabetical order

- Process - This is often used where there is a complicated data structure, rather than when you just need data based on single fields (e.g. owner, joint owner or first life assured on a policy). In this case, the "search" button will offer a popup window of all processes that are available to retrieve data

e.g. ASSIGNEE.CLIENT.KEY.SYMB returns the client number of the policy's assignee.

Label Prefix

Where multiple, similar inherited templates are used, this defines the prefix on the "{ }" delimited variables used in the MS Word template, enabling the same data field to be output for different entities.

e.g. assume we inherit client data for policy owner and first life assured. We could use " owner " and " life1 " as label prefixes:

- this gives us { ownerName } & { life1Name }, and { ownerDOB } & { life1DOB }, assuming Name and DOB are defined as labels in the inherited client template, for use in the document.

Enables the user to add more inherited templates

Label block

Label

Field label used in the MS Word template (and delimited by squiggly brackets - "{ }").

Type

Select from the drop down

- Database field - key value for the inherited template is a field on the record (from field "Table Name") above

- Process - key value for the inherited template is defined in a program.

Name

- Database field. In this case, the "search" button will offer a popup window of all fields on the table, in alphabetical order.

Putting ",n" after the field will return the "n th " instance of a multivalue field

e.g. "PLANS,1" will return the base plan code.

Process - This is often used where you need more than you just "raw" data from a record. In this case, the "search" button will offer a popup window of all processes that can be used.

This feature is often used in conjunction with conditional processing (row-stripping in tables) in Document Server. When Document Server finds the string "@@@@@" in a row in a table in a MS Word template, it deletes the entire row. This offers considerable flexibility in document output, for example allowing alternative wordings depending on policy features such as whether there are one or two owners.

Conversion

Data field conversion. This enables the "raw" data in a field to be converted to the required output format.

e.g.

-@

Returns "@@@@@" if value is false.

If value is true, returns null.

+@

Returns "@@@@@" if value is true.

If value is false, returns null.

-@'x'

Returns "@@@@@" if value is not x

If value is x, returns null.

n.b. you can substitute any other check value for 'x'

+@'x'

Returns "@@@@@" if value is x

If value is not x, returns null.

n.b. you can substitute any other check value for 'x'

+@

Returns @@@@@ if value is null.

If value is not null, returns null.

*@

Returns @@@@@ if value is null.

If value is not null, returns the value.

D4/

converts a date into mm/dd/yyyy (US format)

DV

converts a date into dd/mm/yyyy (European format)

MDn

converts a number to "n" decimal places

Gn*y

Returns the n th part of a multipart key (y represents the number of key parts you want)

e.g. Key on commission statement is

23*GBP*05578

("statement number"*"currency"*"agent number")

G0*1 will return 23

G0*2 will return 23*GBP

G3*1 will return 05578

CCY

returns the value, to the correct number of decimal places for the policy currency, with the appropriate currency prefix or suffix e.g. 712.49.

The currency parameters are defined at Support > Currency > Currency Codes

MCL

Converts data to lower case

MCT

Converts second and subsequent characters in words to lower case

e.g.

JOHN OBI MIKEL is converted to John Obi Mikel

MCU

Converts data to upper case

NR

Converts data to Roman Numerals

#PLANS

Returns plan full name (when data is plan code)

#CURRENCY

Returns currency name (when data is currency code from Support > Currency > Currency Codes)

#COMPANY

Returns company name (when data is company code from Support > Organisation > Company)

#DD_PAYMENT_METHOD

Returns name of direct debit payment method (when data is dd payment method code - see Support > Codes > DD Payment Methods)

#FUNDS

Returns fund name (when data is fund code)

#<xxx>

Returns the code description from CODES record <xxx> (where data is the code value)

e.g. #SEX returns "Male" or "Female", where the data is the code value (M or F) from CODES record SEX.

+n

Adds "n" to a number (or "n" days to a date)

-n

Subtracts "n" from a number (or "n" days from a date)

*n

Multiplies number by n

/n

Divides number by n

+MONTHS<N>

Adds N months to a date

e.g. +MONTHS6 adds six months

-MONTHS<N>

Subtracts N months from a date

e.g. MONTHS3 subtracts three months

+YEARS<N>

Adds N years to a date

e.g. +YEARS5 adds five months

-YEARS<N>

Subtracts N years from a date

e.g. YEARS1 subtracts 1 year

Variables can be used for some conversion

Variables are defined by adding labels in <> brackets

e.g. if you want to deduct a policys suspense balance from the due premium to show a net premium, you can define it as a label <susp> and include it as a conversion -<susp> on the policy premium label (process GET.NEXT.PREMIUM.AMOUNT.

SYMB)

FINDVx

Finding index(position) of the x in multivalued field.

e.g. if field is PLAN_CD, FINDVEPP will return position of plan EPP in the list of plan codes.

Normally assigned to variable, which is later used to retrieve subvalues from related multivalued fields e.g. FACE_AMOUNT,<IndexOfEPP>

In addition, any standard U2 data conversion can be used. More can be found at http://www.webelink.com/WL_info_view.asp?clt=asi&cls=pick_oconv.
