Documentation to be produced

List of document event triggers and the associated standard letter or workflow codes

Label

Description

Printing

Indicates whether documentation will be generated when the events occur (O-Online), or when the end of day batch is run (B-Batch).

If B is selected, document triggers are written to a holding file LIFELISTS when the event occurs, and processed (moved to LIFELISTS_DONE) by batch job LFSDO.IDOCN Policy Documentation.

Event

The event code that, when generated, will trigger the document production.

Language

The documentation language is obtained from the policy, or client, that the event relates to. The document will only be produced if this language code is the same as the events. This allows you to define different documents for each language.

Type

P-Call a Process

Allows a specific program to be attached to the event.

D-Plan Specific

The documentation requirements for the event are set separately for each plan type (uses Product Workbench > Documentation).

C-Company Specific

The documentation requirements for the event are set separately for each company (uses Organisation > Company Documentation).

W-Create DocServer Document

This generates the document set in Merge Name field.

T - Workflow Task

Create a new workflow task

F - Close Workflow Task

Close an open workflow task

K- Keep it post-processing

Used in conjunction with special batch processes where a LIFEfit-style event trigger is required to generate output in a specific format, e.g. a CSV file to be picked up by a third-party application and generate an SMS text message

Merge Name

This is the Standard Letter code (defined using Data > Standard Letters).

For Workflow-related types (T and F above), this is the workflow task code

Alternative Path

Spool path to use for the output file for Document Server, if non-standard (i.e. not the one defined at Admin > Configuration > File Location).

This is often used for secure printing, or printing on specialist stationery e.g. cheque printing

Condition

This limits the production of the document to those policies that match the selected condition.

Options:

       ASS - Assigned Policy only

       GR_C - Group certificates only

       GR_P - Group master policy only

       IND - Individual policy only

       MSEG - Main Segment only

       NON_ASS - Non Assigned policy only

Excl from print

Click the check box if this document should not appear in the list of documents available for printing by manual request at policy workbench.

Often used to exclude documents whose printing process includes accounting transactions (e.g. group list bill which posts premium accruals).

Async

This checkbox appears for events which call a process.

If ticked, the process will be processed asynchronously the start of the following event will not be dependent on this one finishing first.
