Web User (eSurance) Security

eSurance user id details.

eSurance is used by a variety of external stakeholders, each of whom needs access to a restricted set of functionalities. Agents, for example, typically need to be able to produce pre-sales quotes and enter new business applications, whereas policyholders just need enquiry access to their own policy.

Label

Description

Name

eSurance user name, as entered on the previous screen.

Language

Language selection for screens. Where translations existed, they will be used.

Agent No

LIFEfit Agent Code to be for this user. The user will be able to access policies he has sold, assuming the policy owner has approved web access.

Leave blank if not applicable (i.e. this is a policyholder or group scheme owner).

Client No

LIFEfit Client code for this user, if applicable. The user will be able to access policies where he/she is the policy owner.

E-Mail

Enter the user email address.

Allowed Functions

Tick the functions the user is allowed to perform

New Business

New business (online applications)

Policy Enquiry

Enquire on existing policies

Quotations

Perform pre-sales quotations (these are created as PDFs on the users desktop, and saved in DocStore).

Group Business Bills

List bills for group schemes.

Group Business

Pre-sales quote and application functionality for group schemes

Agent loan information

Details of loans made to the agent.

Premium on Agent's Policies

Adds a Download Premiums option to the eSurance Menu. Download Premiums generates a list of Premiums Received for the Agents policies (including policies where the Agent is entitled to override commission).

Web Site

Select the web site the eSurance user connects to

Active

Tick if the user is allowed to log in to eSurance (and untick if access is barred)

Allowed to access data of other agents in hierarchy

For override agents, enables the agent to see policies where he is the override agent.

Allowed IP addresses (if empty - all are allowed)

Enables access to be restricted to connection from a specified IP address

<Add> button

Add another allowed IP address

SSL Client Certificate Serial Number

Where SSL is used, enables control by security certificate.

<Save> button

Save changes.

<Sensitive Information> button

Password-protected access to further user profile parameters:

-        eSurance administrator authority flag

-        allowed to access data of other agents in hierarchy (for override agents, enables the agent to see policies where he is the override agent) .

-        Third Party Administration Functionality

<Password Enquiry> button

Password-protected access to view/change the eSurance password.

<Change Password> button

Allows the password to be changed and automatically emailed to the user, without being displayed on screen.

<Return> hyperlink

Return to the main menu.
