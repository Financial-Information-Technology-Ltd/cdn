Menu

Default menu for members of the group.

Allowed Batches

Select from the dropdown of batches defined at Admin > Security > Batch setup.

Users will be presented with these options when at Admin > Run Batch.

Payments Out Limits

Currency

Display only active currencies on the system are shown.

Payout limit

Maximum payment out amount that a member of the group can initiate, and will be paid without requiring second user authorisation.

If set to zero, all payments will require second user authorisation.

Payout authorisation limit

Maximum payment out amount that a member of the group can authorise, where the payment was initiated by someone else.

Typically a low amount is set for clerical users, and higher amounts for supervisors and senior managers, depending on company rules

e.g.

claim administrator can authorise up to 10,000

claims supervisor can authorise up to 50,000

operations director can authorise up to 250,000

managing director can authorise up to 5,000,000

n.b. LIFEfit maintains an audit trail of who initiated a claim, and who authorised it (including time and date of action).

Allowed assign workflow task

Tick if members of the group are allowed to assign (and reassign) workflow to others.

Typically only ticked for managers and supervisors.

Allowed change workflow dates

Tick if members of the group are allowed to change workflow target dates.

Typically only ticked for managers and supervisors.

Allowed to see history of a workflow task

Tick if members of the group are allowed to see the history of a piece of workflow.

Restrict Issue Same User

Tick if members of the group are not allowed to make a policy active if they were also the last person to make a change to the policy.

This is sometimes referred to as a 4-eye check on new business.

Allow commission override

Tick if members of the group are allowed to override the default commission rate for a product at new business, for products where this is allowed. Typically only permitted for members of supervisory or management groups.

Path to Document Server directory (if user group - specific)

Enables a printer location to be user-group specific.

This is useful where different documents are printed by different categories of user in different locations.

Backdate limit for issuing policies (days)

Limits how far policy issue can be backdated by members of the user group.

Typically, supervisors and managers will be able to backdate further.

Categories of the documents users are allowed to access

LIFEfit has functionality to restrict access to DocStore items. This is achieved by setting a document category and sensitivity on each document. For outgoing, system-generated documents, this is defined as part of the setup (Support > Data > Standard Letters). For incoming documents, it is defined on upload. A document's sensitivity can be modified at Admin > User Adjustments > Move DocStore Item.

n.b. if left blank, all categories of document at all levels of sensitivity can be accessed
