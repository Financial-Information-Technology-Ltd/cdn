Set of Questions

These are the questions, included at the end of new business entry and the eSurance application, which assess the applicants health and lifestyle. They form the basis of the underwriters decision to accept an application on standard terms, apply a rating (treat as a substandard life) or decline the application.

The set of questions to be used in the new business screens for a product is defined at Product Workbench, Medical Questionnaire.

The question sets defined here are also used for financial underwriting questions which are completed during the application / new business process, configured at Product Workbench, Financial Questionnaire. These are aimed at ensuring the client is not being missold a policy (needs analysis and ability to pay), as well as compliance issues such as establishing source of funds.

For a policy, the questions asked and answers entered as part of the application / new business process can be seen from the answers to questions hyperlink at the bottom of the policy workbench.

Label

Description

Code

The key for this set of questions, as selected or entered on the previous screen. Display only.

Name

Free format text description of the question set.

Multiple Lines

Check if multiple lines are to be entered.

This can be used, for example, if it is necessary to list an unrestricted number of beneficial owners and the percentages they own, or to list company directors to be included in a key man policy.

Additional lines are added by clicking the + button, as offered

Use Template

Check the box if a pre-defined html template is to be used. This is used where the standard, single column input format is not sufficient (e.g. if you want two columns for entering owner and spouse responses to questions).

Template Name

If Use template is checked, select the template to use from the dropdown list.

Triggered Underwriting Requirement

Select the underwriting code (if any) which should be applied to the policy if one or more of the trigger answers is set to yes or no, as specified below.

The list is comprised of underwriting requirements defined at Support > Underwriting > Requirements.

This functionality assists in referring cases to the underwriters workflow queue if necessary, and enabling clean applications to proceed to issue without underwriter involvement.

Main section

Text

Enter the text to be displayed for the underwriting question

e.g. Have you ever suffered from any heart related disease?

Type

Select the reply format from the dropdown list

       Yes/No (radio button input with yes first)

       No/Yes (radio button input with no first)

       Text

       Header (no input required text is displayed with a bold background bar)

       Integer

       Monetary Amount (number with two decimal places)

       Number

       Choice (user will select from a dropdown list, from a CODES record selecting this option opens up a text box for the name of the CODES record to be entered)

       Formula (not available if either multiple lines or use template is selected. Opens us a text box)

Trigger answer

For yes/no questions, select which option should trigger the underwriting requirement defined in Triggered Underwriting Requirement above.

e.g.

       if the question is Have you been referred to a hospital for tests? , yes would be the trigger answer.

       if the question is Are you in good health? , no would be the trigger answer.

If not applicable (i.e. no need to automatically set an underwriting requirement), leave blank.

If Yes

For radio button options, defines what should happen if yes is selected.

Select from the dropdown list of underwriting question sets (allows nesting of questions).

e.g. if the user enters yes to the question Have you ever suffered from any heart related disease? , a follow-up question Please provide full details appears (as defined in a separate set of questions).

Optional

Tick if the question does not need to be answered in order to proceed with an application at new business.

If unticked, a warning will appear on screen when attempting to proceed to the next screen without answering. The user will not be able to proceed until the question has been answered.

<Edit> button

Edit an existing question.

<Insert> button

Insert a new question in the list.

n.b. it is not a good idea to insert a question into a list which is already in production. When enquiring on the question set for policies created using the previous definition, the questions and answers will no longer line up. Instead consider creating a new question set and associating that with the product.

<Delete> button

Delete a question from the list.

n.b. it is not a good idea to delete a question from a list which is already in production. When enquiring on the question set for policies created using the previous definition, the questions and answers will no longer line up. Instead consider creating a new question set and associating that with the product.

<Add> button

Add a question to the end of the list

<Save> button

Save the set of questions

<Return> hyperlink

Return to the main menu without saving
