Cash Payment Types

Define the attributes of a payment in type (e.g. cash, cheque, credit note). On registering a payment on LIFEfit, the user specifies what type of payment it is and the system treats it according to the rules defined here.

Label

Description

Name

The description for the selected Cash Payment Type code.

Days to Clear

The number of days to be added to the effective date when calculating the date that the payment will be considered to be cleared.

Use original effective date when reconciling transactions

This parameter controls the effective date of reconciled payments.

When set, the effective date will be set to the payments original effective date.

When un-set, the effective date will be set to the systems effective date.

This is required so that the policy holder is not dis-advantaged by a delay in reconciling the payment (for example a loss of interest on deposit based policies).

List Type

Enter the list type (document trigger) to be generated for payments of this type.

This will be the value in the first column at Admin > Config > Documentation (document trigger setup).

If left blank, a default List Type CASH_RECEIPT will be generated.

Pseudo Group

If applicable, select a pseudo group from the dropdown list.

This is used where the cash pseudo (i.e. bank account) on which a payment has been received is identified at the point it is receipted, either using the online receipting screen or by calling the payPolicy webservice. On the online receipting screen, the user selects from a dropdown of the pseudos in this pseudo group. In the webservice, the pseudo is one of the parameters passed to LIFEfit.

Cash Pseudo

The default accounting pseudo code to be used for this payment type.

Currency

The currency code for this payment type. Leave blank if multi-currency and the currency code will be provided with the payment details.

Languages

Allows the Name value to be set for other Languages.

Branches

Select the Branches that will be allowed to use this payment type.
