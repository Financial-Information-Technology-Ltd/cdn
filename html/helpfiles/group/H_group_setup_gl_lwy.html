Scheme Setup

This screen shows information about the Group Life Scheme.

It allows the user to perform servicing tasks on the scheme

Label

Description

Scheme number

Group scheme number

Group Owner

Client number and name of group scheme owner

Number of members

Number of members in the scheme

Scheme status

Scheme status

Base Plan

Base plan (product) code and name

Scheme RCD

Scheme risk commencement date

External Ref

External reference number sometimes used to hold the scheme number for groups originally administered elsewhere.

Policy Number

Group master policy number.

This is a hyperlink click it to open the group master policy in policy workbench.

Currency

Scheme currency

Mode method

Premium payment method and frequency

Proposal Serial Number

Number of the original scheme proposal (if entered during the scheme setup / new business process)

Documentation Language

Scheme documentation language

Risk Category

Scheme risk category (industry sector).

This is one of the parameters used by LIFEfit to read the premium rate. It enables higher rates to be used for risky industries, such as construction or oil & gas.

Rate Age Adjustment

Age adjustment to be included for all scheme members.

Negative numbers use younger ages, positive numbers use older ages

One way this is used is to enable schemes to be renewed on last years rates by entering -1

Coinsured Scheme

Indicates whether the scheme is coinsured.

Lead Scheme

Group number and name of the lead scheme (parent company scheme), if any.

Schemes are administered separately, and this enables parent company information to be included on documentation for this scheme.

Renewal period (also known as pro rata period)

Where the scheme is annual, but the first renewal is in less than one year, enter the number of months until renewal here.

First premiums will be pro-rated i.e. n/12 of a full annual premium, where n is the renewal period. At renewal, a full annual premium will be billed.

If left blank, a full annual premium will be billed at scheme commencement.

Amount in suspense

Amount of money held in suspense on the group master policy. This amount is available to pay certificate premiums.

Base Coverage Configuration

This section displays the rules for the base, group life coverage

Sum Assured Basis

Multiple of Salary or Fixed Amount, as entered at new business

Sum Assured Value

If a multiple of salary scheme, the salary multiple

If a fixed amount scheme, the amount.

Unit Rate

Unit rate used in premium calculation, if entered at new business.

Rates are stored as rate per hundred. If you are expecting to see a rate per mille, remember to divide by 10.

e.g. a rate of 4.36 per mille is shown as 0.436.

If blank, an age-specific premium rate table is used. This premium rate is defined in plan rules (product workbench > premiums premiums)

Adjustment Percent

Shows the percentage adjustment (loading/discount) to the premium rate (or unit rate) used for all scheme members.

A discount is shown as a value less than 100 e.g. a 15% discount shows as 85

A loading is shown as a value greater than 100 e.g. a 10% loading shows as 110.

Initial Commission

If the scheme uses a commission rate which differs from the default for the plan, it is shown here

Free Cover Limit

The default free cover limit (FCL) for the scheme

This value may be overridden at subgroup (employee category) level

Schemes can be issued with members in underwriting. If the underwriting is not cleared a death claim will be capped at FCL.

Facultative Reinsurance Treaties

If the scheme is reinsured on a facultative basis, treaty details are shown here.

Please note that all the group scheme servicing buttons can be controlled (switched on or off) as part of the users menu configuration (Administration > Security/Processes > Menu Details)

<Show Members> button

Click to list scheme members

<Calculate Bill> button

Generates a bill in CSV format based on current membership information. For newly added members, a certificate (policy) record is created in this process

<Pay Premium on Certificates> button

Attempts to pay certificate premiums from suspense on the master policy, according to the most recent bill

<Accounting> button

Takes you to the policy accounting journal screen for the group master policy

<Import Members> button

Import member information from CSV file

<Reverse Last Premium> button

Reverse the most recent premium

<Change Status> button

Change group scheme status

<Calculate Leavers/Joiners Bill> button

Calculates an ad hoc bill for changes in membership information pro rata premium for new joiners, pro-rata refund for leavers, and additional premium for ratings (if any)

<Pay Leavers/Joiners> button

Pay premiums according to most recent leavers/joiners bill

<Pay Partial Premium> button

This button is visible if there is money in suspense on the group master policy.

It gives you a screen where the suspense amount can be allocated to individual member certificates, up to the total suspense balance.

<Input Payment> button

Takes you to the cash receipting screen to enter details of a payment received

<Underwriting> button

Takes you to the screen for underwriting requirements for the group master policy. You can add new underwriting, or update the details of existing underwriting requirements (e.g. clear them)

<Print Leavers/Joiners Bill> button

Print ad hoc bill for changes in membership information pro rata premium for new joiners, pro-rata refund for leavers, and additional premium for ratings (if any)

<Medical Provider> button

Takes you the screen to identify the clinic dealing with medical checks for scheme members. Any notifications of medical tests required as part of the underwriting process will be sent (or emailed) here. Payment for tests will be made to this clinic.

<Reproduce Bill CSV> button

Generates a CSV version of the most recent invoice.

Can be found in the &hold& folder

<Loss Ratio> button

Displays the loss ratio for each scheme year (total of claims processed and pending DIVIDED BY total of premiums paid).

Values are displayed as a percentage e.g. 50,000 claims for premiums of 1,200,000, loss ratio is 4.17%

These values are populated by batch job LF.GROUP.LOSS.RATIO which should be run on a regular basis, possibly weekly, in order to keep values for the current year up to date. The screen shows when the value was last updated.

<Return> hyperlink

Returns you to the previous screen
