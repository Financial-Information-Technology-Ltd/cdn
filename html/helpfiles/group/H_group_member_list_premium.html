Scheme Members

This screen lists the general details of all the members of the group. The list can be sorted in ascending/descending or alphabetical order according to the Member Number, Name and Date of Birth. This screen also allows you to edit or hold existing members and add new members to the scheme.

Label

Description

Scheme Number

Group Number which is a unique identifier of the group.

Scheme Owner

Group Client Name and Corporate Client Number.

No

This is the members number which can be sorted.

Name

Full Name of the Group Member which can be found on the member record.

Subgroup

This displays which subgroup the member belongs in. The Subgroup Field on scheme member screen is only displayed if there are 2 or more subgroups set up in the Group Scheme.

Birth Date

Date of Birth of the member which can be found on the member record.

Premium

This displays total modal premium the member is to pay.

Certificate

This is the Certificate Number which can also be regarded as a policy number is a hyperlink to the Policy Details screen. The Certificate Key Structure can be defined at Plan Level ( Plans > Group Business> Certificate Key Structure ). Certificates are generated when the Group Bill is calculated for the first time or in the case of new joiners, when calculating Leavers and Joiners bill.

Like Policy Number, the Certificate is identified as a unique foreign key and can also be searched for using the Policy Workbench screen.

Status

Current Status of the Members Certificate.

Start Date

This is the date from which the member becomes active in the group. On the Policy Certificate this will be the issue date.

Paid to Date

This is the date at which premium is paid up until. The next premium is due on this date.

Changes Pending?

If a member wishes to change the details of his/her certificate for example change in salary or additional dependant. This change will be stored as pending and flagged on the member list as Yes alongside the member record.

Pending Changes made prior to premium due date will get cleared once the premium is received. If the Pending change occurs after the premium due date an MC future record is created and once processed in the batch will clear the pending changes flag.

Hold

The Hold option, if checked will exclude a member from any future group invoicing.

Edit

The edit button will open up the member screen which includes all the details of the member which can be updated.

Add Member

This button opens up a scheme member entry screen which allows you to enter available details of a new joiner. Once saved a member record is created. A Certificate number is assigned once the Total Bill or Adhoc Bill is calculated for the group.
