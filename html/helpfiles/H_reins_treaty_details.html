Reinsurance Treaty

Defines reinsurance treaty arrangements.

The treaty is an agreement between the insurance company and the reinsurer. It defines how the reinsurance premium is calculated, how reinsurance commission (rebate) is dealt with, and how a reinsurance claim is treated when a policy claim event (e.g. death) occurs.

One or more reinsurance treaties can be associated with a plan code (Product Workbench > Reinsurance). The same treaty can be used for multiple plans.

Policies whose plan code(s) have a reinsurance treaty will have REINS future record created at issue for the first reinsurance premium calculation. When processed in batch, this generates a record in table REINSURANCE which can be reported using Davywriter or via ODBC.

Lower Limit

Value below which the insurance company retains the risk

Or percent of total SAR whichever lower

Set the lower limit to this percentage of total sum at risk, if lower than the amount entered above

Upper Limit

Value above which the risk is not covered under this treaty.

There is often another treaty which defines how sums at risk above this are reinsured.

Minimum Reinsured SAR

Allows you to avoiding calculating (and charging) negligible reinsurance premiums. i.e. if SAR (sum at risk) is only just above a treatys lower limit.

Reinsurance company share (%)

Percentage of the sum at risk between the lower and upper limits that is ceded (passed on) under this treaty.

There can be multiple treaties, allowing multiple reinsurers to each take a proportion of the risk. Moreover, if this does not total 100%, the insurance company itself retains some of the risk.

Frequency

Frequency of reinsurance premium calculation. Defines when a subsequent REINS future record is created.

Incidence

Advance or arrears.

If advance, the first REINS future record is created on policy issue date. If arrears, it is created at issue date PLUS 1 frequency (above).

SAR calc code

Sum at Risk Calculation Code

1. Total Sum at Risk: sum at risk is totalled for all policies with the same life assured to determine how the retention is apportioned.

2. Cumulative Sum at Risk: sum at risk is cumulated for all policies with the same life assured so far processed to determine which treaty applies

Reinsurance Premium Rate table

Rate table used to calculate reinsurance premium. Rate table structure is defined at Support > Rates > Rate Structure. Rates are entered / imported at Support > Rates > Rate Entry.

n.b. reinsurance premium = (Reinsured Sum at Risk * Rate)

If premium rate is overridden on policy/group, use it

Usually used for group life business which is sold using a negotiated unit rate, rather than a normal (e.g. age/sex/smoker status dependent) premium rate table.

If the certificate (policy) has its own unit rate, defined as part of the scheme or subgroup rules, use this instead of the Reinsurance Premium Rate Table defined above for the treaty.

Reinsurance rate (%)

Allows a fixed premium rate (if not read from a table)

Rebate rate table

Reinsurance rebate (also known as reinsurance commission) table. Rate table structure is defined at Support > Rates > Rate Structure. Rates are entered / imported at Support > Rates > Rate Entry.

Linked reinsurance treaty

Allows you to link treaties together, where there are related risks on a product.

This is typically used where there is both critical illness and life cover, to ensure the excess cover only (i.e. amount by which life cover exceeds critical illness cover) is reinsured under this treaty.

Reverse reinsurance in case of surrender or lapse

Enables reinsurance premium to be automatically clawed-back from the reinsurer in the event of a lapse or surrender (often used where reinsurance premium is calculated annually in advance).

Number of years before recapture

LIFEfit supports automatic treaty recapture (change of reinsurance treaty).

e.g. Reinsurer A offers a recapture option after 15 years. Therefore once a policy has been administered for 15 years, it can then be moved to Reinsurer B.

Recapture reinsurance treaty

Select the recapture (replacement) reinsurance treaty from the dropdown list.

Days before treaty recapture notification produced

Number of days before the recapture date that a letter is triggered by the processing of a RECAP future record.

The number of days creates a future record with the reason EVENT, n days before the recapture date, with a LIST_TYPE of "TREATY_RECAPTURE_NOTICE".

The RECAP future record is processed by batch job LF.TREATY.RECAPTURE which produces batch report LF008 in the &hold& folder.
