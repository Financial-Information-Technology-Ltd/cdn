Workflow task codes

 

Select an existing workflow task code, or create a new one. A workflow task code represents a category of work that takes place in the back office, and needs to be monitored.

 

Workflow can be initiated in multiple ways

·         Manually

·         By system event (document trigger) e.g. on status change to lapse, there may be a requirement to chase the policyholder for overdue premium

·         Automatically by incoming email (often used for policyholder or agent queries)

·         Webservice (often used for “contact us” option on company mobile app or website)

·         Chained on completion of previous workflow

·         Scheduled (weekly, monthly, quarterly, annually) and created in batch when due
