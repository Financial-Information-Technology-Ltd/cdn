Interest Recalculation

This screen is used to recalculate savings interest on deposit administration (DA) savings policies.

The actual interest rate to be paid on DA policies is often declared at year end, but an interim rate is used in the mean time as an approximation. This ensures that any claims (e.g. surrender, death or maturity) receive an amount including interest credit.

Once the actual interest rate is declared, and the interest rate table updated, this screen is used to post an accounting adjustment (and policy value update) to all DA interest postings for the period identified.

The recalculation can be limited by company, plan code or individual policy if required.

Interest rate tables are maintained at Support > Rates > Interest Rates.

The interest rate table applicable to a product is set at Support > Product Workbench, Deposit Administration.

Label

Description

Start Date

Date from which all interest postings will be adjusted (additional posting made for the difference between interim rate used and actual rate).

End Date

Date to which all interest postings will be adjusted.

Policy:

If entered, interest will be recalculated for this policy only. Useful if adjustments or backdated premiums necessitate recalculation (will process much quicker for on epolicy than for all).

If left blank, all policies will be processed.

Plan Code:

If you wish to restrict the calculation to policies of one plan code only, please select form here.

If left blank, policies of all plan codes will be processed.

Company:

Select a company code from the dropdown list if you wish to restrict the recalculation by company. Useful if you are administering business for multiple companies and final interest rates are declared at different times.

If left blank, policies for all companies will be processed.
