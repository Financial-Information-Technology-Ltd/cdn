NL Account Input vs LF Accounts

Enter the nominal ledger account details for the LIFEfit account number selected on the previous screen. This includes the analysis codes which enable you to slice and dice general ledger accounting information in the corporate GL package (e.g. Sun Accounts).

An example of this is where you may want to look at premim received at a consolidated level, or split it out by product type, policy branch or writing agent area.

Label

Description

Account

LIFEfit account number, as selected on the previous screen. Display only.

Company

Company code of the branch selected on the previous screen. Display only.

Branch

Branch code, as selected on the previous screen. Display only.

Currency

Currency, as selected on the previous screen. Display only.

NL Account

Enter the nominal ledger (general ledger) account number.

Free format text

Item Transfer

Select how accounts should be produced in the interface file from the dropdown list. Options are

       I - Itemised with Policy No

       T - Itemised with Transaction No

       S Summary

Analysis Code 1 to Analysis Code 10

Select the data you wish to be output as analysis codes with each account entry. Options are listed below

$AGENT - policy writing agent

$AREA - agent area

$BRANCH - policy branch

$COUNTRY - policy country

$EXTERNAL_POL_NO - policy external reference number

$FUND - fund code

$FUND_ISIN - fund code's ISIN (International Securities Identification Number)

$FUND_TYPE - fund type

$INTERNAL.REF.NO - agent's internal reference number

$LAW_COUNTRY - policy country of law

$PLAN - plan code

$PLAN_CLASS - plan class type

$POLICY - policy number

$PRODTYPE - If this is a dedicated fund product (PPB), "P"

$PSEUDO - LIFEfit account pseudo

$REFERENCE - Field "reference" on the accounting record

$REGION - agent region

$SOURCE - account source (ADJ means it was an adjustment posting)
